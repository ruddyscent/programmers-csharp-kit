# 🥊 순위 — C# 학습자료

## 📌 1. 문제 핵심

권투선수 `n`명이 있고, 일부 경기 결과가 주어집니다.

경기 결과는 다음 의미입니다.

```text
A가 B를 이겼다
```

이때 정확하게 순위를 알 수 있는 선수의 수를 구해야 합니다. 🎯

어떤 선수의 순위를 정확히 알 수 있으려면, 그 선수가 다른 모든 선수와 비교 가능해야 합니다.

즉, 선수 `x`에 대해 다른 모든 선수 `y`와 다음 둘 중 하나를 알아야 합니다.

```text
x가 y보다 강하다
y가 x보다 강하다
```

---

# 🔍 예제 이해하기

```text
n = 5
results = [
  [4, 3],
  [4, 2],
  [3, 2],
  [1, 2],
  [2, 5]
]
```

직접 주어진 결과만 보면 일부 관계만 알 수 있습니다.

하지만 간접 관계도 알 수 있습니다.

예를 들어:

```text
4가 2를 이김
2가 5를 이김
```

이면:

```text
4는 5보다 강하다
```

라고 판단할 수 있습니다.

2번 선수는 `1`, `3`, `4`에게 지고, `5`를 이깁니다.

그래서 다른 모든 선수와의 관계를 알 수 있습니다.

5번 선수도 다른 모든 선수에게 지는 관계를 알 수 있습니다.

따라서 정확한 순위를 알 수 있는 선수는 2명입니다. ✅

---

# 🧠 핵심 아이디어: 승패 관계 전파하기

이 문제는 그래프처럼 볼 수 있습니다.

```text
A가 B를 이김 → A에서 B로 가는 방향 간선
```

만약 다음 관계가 있다면:

```text
A → B
B → C
```

그러면:

```text
A → C
```

도 알 수 있습니다.

즉, 직접 경기하지 않았더라도 중간 선수를 통해 승패 관계를 추론할 수 있습니다. 🔗

이렇게 모든 간접 승패 관계를 채운 뒤, 각 선수가 다른 모든 선수와 비교 가능한지 세면 됩니다.

---

## ⚠️ 왜 모든 쌍의 관계를 봐도 괜찮을까?

선수 수 `n`은 최대 100명입니다.

따라서 `n³` 알고리즘도 충분히 가능합니다.

```text
100 × 100 × 100 = 1,000,000
```

이 정도는 코딩 테스트에서 부담 없는 크기입니다. 😊

---

# 🧠 풀이 1. 이해하기 쉬운 방법 — 플로이드-워셜로 승패표 채우기

## 💡 아이디어

1. `win[a, b] = true`는 `a`가 `b`를 이긴다는 뜻입니다.
2. 주어진 경기 결과를 `win` 배열에 저장합니다.
3. 중간 선수 `mid`를 거쳐 알 수 있는 승패 관계를 채웁니다.
4. 각 선수마다 다른 모든 선수와 비교 가능한지 확인합니다.
5. 비교 가능한 선수를 정답에 더합니다.

---

## 💻 C# 코드

```csharp
public class Solution
{
    public int solution(int n, int[,] results)
    {
        bool[,] win = new bool[n + 1, n + 1];

        for (int i = 0; i < results.GetLength(0); i++)
        {
            int winner = results[i, 0];
            int loser = results[i, 1];

            win[winner, loser] = true;
        }

        for (int mid = 1; mid <= n; mid++)
        {
            for (int from = 1; from <= n; from++)
            {
                for (int to = 1; to <= n; to++)
                {
                    if (win[from, mid] && win[mid, to])
                        win[from, to] = true;
                }
            }
        }

        int answer = 0;

        for (int player = 1; player <= n; player++)
        {
            int known = 0;

            for (int other = 1; other <= n; other++)
            {
                if (player == other)
                    continue;

                if (win[player, other] || win[other, player])
                    known++;
            }

            if (known == n - 1)
                answer++;
        }

        return answer;
    }
}
```

---

## 🔎 코드 해설

### 1. 승패표 만들기

```csharp
bool[,] win = new bool[n + 1, n + 1];
```

`win[a, b]`가 `true`라면 `a`가 `b`보다 강하다는 뜻입니다.

선수 번호가 1부터 시작하므로 배열 크기를 `n + 1`로 만듭니다.

---

### 2. 직접 경기 결과 저장

```csharp
win[winner, loser] = true;
```

주어진 결과 `[A, B]`는 `A`가 `B`를 이겼다는 뜻입니다.

따라서 `win[A, B]`를 `true`로 바꿉니다.

---

### 3. 간접 승패 관계 전파

```csharp
if (win[from, mid] && win[mid, to])
    win[from, to] = true;
```

`from`이 `mid`를 이기고, `mid`가 `to`를 이긴다면:

```text
from은 to보다 강하다
```

라고 알 수 있습니다.

이 과정을 모든 중간 선수에 대해 반복하면 가능한 모든 승패 관계가 채워집니다. 🧠

---

### 4. 순위를 알 수 있는 선수 확인

```csharp
if (win[player, other] || win[other, player])
    known++;
```

`player`와 `other` 사이의 관계를 알 수 있으면 `known`을 증가시킵니다.

다른 `n - 1`명과 모두 비교 가능하다면 이 선수의 순위는 정확합니다.

```csharp
if (known == n - 1)
    answer++;
```

---

## ⏱️ 시간 복잡도

선수 수를 `n`, 경기 결과 수를 `m`이라고 하겠습니다.

경기 결과를 저장하는 데:

```text
O(m)
```

승패 관계를 전파하는 데:

```text
O(n³)
```

마지막으로 각 선수의 비교 가능 여부를 확인하는 데:

```text
O(n²)
```

전체 시간 복잡도는:

```text
O(n³)
```

입니다.

---

## 📦 공간 복잡도

`bool[,] win` 배열을 사용합니다.

```text
O(n²)
```

입니다.

# 🚀 풀이 2. 짧은 코드 버전 — 승패표를 바로 갱신하기

## 💡 아이디어

풀이 1과 같은 플로이드-워셜 방식입니다.

다만 변수 이름을 줄이고, 승패 전파를 한 줄로 표현합니다.

```csharp
win[i, j] |= win[i, k] && win[k, j];
```

`i`가 `k`를 이기고 `k`가 `j`를 이기면 `i`가 `j`를 이긴다고 표시합니다. ⚡

---

## 💻 C# 코드

```csharp
public class Solution
{
    public int solution(int n, int[,] results)
    {
        bool[,] win = new bool[n + 1, n + 1];

        for (int i = 0; i < results.GetLength(0); i++)
            win[results[i, 0], results[i, 1]] = true;

        for (int k = 1; k <= n; k++)
            for (int i = 1; i <= n; i++)
                for (int j = 1; j <= n; j++)
                    win[i, j] |= win[i, k] && win[k, j];

        int answer = 0;

        for (int i = 1; i <= n; i++)
        {
            int count = 0;

            for (int j = 1; j <= n; j++)
                if (i != j && (win[i, j] || win[j, i]))
                    count++;

            if (count == n - 1)
                answer++;
        }

        return answer;
    }
}
```

---

## ✨ 코드 의미

### 1. 승패 전파를 한 줄로 작성

```csharp
win[i, j] |= win[i, k] && win[k, j];
```

`|=`는 기존 값이 `true`이면 그대로 `true`를 유지합니다.

새로운 간접 승패 관계를 찾으면 `true`로 바꿉니다.

---

### 2. 비교 가능한 선수 수 세기

```csharp
if (i != j && (win[i, j] || win[j, i]))
    count++;
```

`i`가 `j`를 이기거나, `j`가 `i`를 이기면 두 선수의 순서를 알 수 있습니다.

자기 자신은 비교 대상이 아니므로 `i != j`를 함께 확인합니다.

---

## ⏱️ 시간 복잡도

플로이드-워셜 방식으로 모든 선수 쌍의 관계를 전파합니다.

```text
O(n³)
```

입니다.

---

## 📦 공간 복잡도

승패표를 저장합니다.

```text
O(n²)
```

입니다.

# 🏆 추천 풀이

처음 공부할 때는 **풀이 1번**을 추천합니다.

이유는 다음과 같습니다.

```text
1. 승패 관계가 어떻게 전파되는지 잘 보입니다.
2. 정확한 순위를 판단하는 기준이 명확합니다.
3. 플로이드-워셜의 핵심 구조를 익히기 좋습니다.
```

코딩 테스트 제출용으로 코드 길이를 우선한다면 **풀이 2번**이 더 좋습니다.

다만 `|=` 표현이 낯설다면 풀이 1번으로 먼저 이해하는 것이 안전합니다. 🌱

# ✅ 최종 정리

이 문제의 핵심은 다음입니다.

```text
직접 승패뿐 아니라 간접 승패까지 알아야 한다.
```

예를 들어:

```text
A가 B를 이기고, B가 C를 이기면 A는 C보다 강하다.
```

이 관계를 모든 선수 쌍에 대해 채웁니다.

그 뒤 각 선수가 다른 모든 선수와 비교 가능한지 확인하면 됩니다.

🥊 “순위”는 플로이드-워셜로 관계의 전이성을 채우는 대표적인 그래프 문제입니다.
