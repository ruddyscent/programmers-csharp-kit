# 🏠 도둑질 — C# 학습자료

## 📌 1. 문제 핵심

집들이 동그랗게 원형으로 배치되어 있습니다.

각 집에는 돈이 들어 있고, 인접한 두 집을 털면 경보가 울립니다. 🚨

즉, 다음 조건을 지켜야 합니다.

```text
서로 이웃한 집은 동시에 털 수 없다.
```

목표는 다음입니다.

```text
훔칠 수 있는 돈의 최댓값 구하기
```

---

# 🔍 예제 이해하기

```text
money = [1, 2, 3, 1]
```

집이 원형으로 배치되어 있으므로 다음 집들이 서로 이웃합니다.

```text
0번 집 ↔ 1번 집
1번 집 ↔ 2번 집
2번 집 ↔ 3번 집
3번 집 ↔ 0번 집
```

가능한 선택 중 하나는:

```text
0번 집 + 2번 집 = 1 + 3 = 4
```

입니다.

`1번 집 + 3번 집`은:

```text
2 + 1 = 3
```

입니다.

따라서 정답은:

```text
4
```

✅

---

# ⚠️ 어려운 점: 원형 구조

일반적인 일렬 집 문제라면 다음처럼 풀 수 있습니다.

```text
현재 집을 털거나
현재 집을 털지 않거나
```

하지만 이 문제는 집이 원형입니다.

그래서 첫 번째 집과 마지막 집이 서로 이웃합니다.

```text
첫 번째 집과 마지막 집은 동시에 털 수 없다.
```

이 조건 때문에 단순한 일렬 DP를 그대로 쓰면 안 됩니다. 🚫

---

# 🧠 핵심 아이디어

원형 문제는 두 경우로 나누면 일렬 문제로 바꿀 수 있습니다.

## 경우 1. 첫 번째 집을 터는 경우

첫 번째 집을 털면 마지막 집은 털 수 없습니다.

따라서 범위는:

```text
0번 집부터 n-2번 집까지
```

입니다.

## 경우 2. 첫 번째 집을 털지 않는 경우

첫 번째 집을 제외하면 마지막 집은 털 수 있습니다.

따라서 범위는:

```text
1번 집부터 n-1번 집까지
```

입니다.

두 경우의 최댓값 중 큰 값을 선택하면 됩니다. 🎯

```text
정답 = max(
    0번 ~ n-2번 집에서의 최댓값,
    1번 ~ n-1번 집에서의 최댓값
)
```

---

# 🧩 일렬 집 털기 DP

일렬로 집이 있다고 생각해 봅시다.

각 집에서 선택지는 두 가지입니다.

```text
1. 현재 집을 턴다.
2. 현재 집을 털지 않는다.
```

현재 집을 털면 바로 이전 집은 털 수 없습니다.

따라서 점화식은 다음입니다.

```text
dp[i] = max(dp[i - 1], dp[i - 2] + money[i])
```

의미는 다음과 같습니다.

```text
dp[i - 1]              → 현재 집을 털지 않는 경우
dp[i - 2] + money[i]   → 현재 집을 터는 경우
```

둘 중 더 큰 값을 선택합니다. 💰

---

# 🧠 풀이 1. 이해하기 쉬운 방법 — 배열 DP

## 💡 아이디어

1. 첫 번째 집을 포함하는 범위 `[0, n - 2]`를 계산합니다.
2. 첫 번째 집을 제외하는 범위 `[1, n - 1]`를 계산합니다.
3. 두 결과 중 큰 값을 반환합니다.
4. 각 범위는 일렬 집 털기 DP로 계산합니다.

---

## 💻 C# 코드

```csharp
using System;

public class Solution
{
    public int solution(int[] money)
    {
        int n = money.Length;

        int case1 = RobLine(money, 0, n - 2);
        int case2 = RobLine(money, 1, n - 1);

        return Math.Max(case1, case2);
    }

    private int RobLine(int[] money, int start, int end)
    {
        int length = end - start + 1;
        int[] dp = new int[length];

        dp[0] = money[start];

        if (length == 1)
        {
            return dp[0];
        }

        dp[1] = Math.Max(money[start], money[start + 1]);

        for (int i = 2; i < length; i++)
        {
            int currentIndex = start + i;

            dp[i] = Math.Max(dp[i - 1], dp[i - 2] + money[currentIndex]);
        }

        return dp[length - 1];
    }
}
```

---

## 🔎 코드 해설

### 1. 두 경우로 나누기

```csharp
int case1 = RobLine(money, 0, n - 2);
int case2 = RobLine(money, 1, n - 1);
```

`case1`은 첫 번째 집을 고려하는 대신 마지막 집을 제외합니다.

`case2`는 첫 번째 집을 제외하고 마지막 집을 고려합니다.

---

### 2. 일렬 DP 함수

```csharp
private int RobLine(int[] money, int start, int end)
```

`start`부터 `end`까지 일렬로 놓인 집이라고 생각하고 최댓값을 구합니다.

---

### 3. 첫 번째 값 초기화

```csharp
dp[0] = money[start];
```

범위 안에서 첫 번째 집만 있을 때는 그 집을 터는 것이 최댓값입니다.

---

### 4. 두 번째 값 초기화

```csharp
dp[1] = Math.Max(money[start], money[start + 1]);
```

두 집 중 하나만 털 수 있으므로 더 큰 값을 선택합니다.

---

### 5. 점화식

```csharp
dp[i] = Math.Max(dp[i - 1], dp[i - 2] + money[currentIndex]);
```

현재 집을 털지 않으면 `dp[i - 1]`입니다.

현재 집을 털면 이전 집은 못 털기 때문에 `dp[i - 2] + money[currentIndex]`입니다.

둘 중 큰 값을 고릅니다. 🧠

---

## ⏱️ 시간 복잡도

각 경우마다 집을 한 번씩 확인합니다.

```text
O(n)
```

입니다.

---

## 📦 공간 복잡도

DP 배열을 사용합니다.

```text
O(n)
```

입니다.

# 🚀 풀이 2. 코드가 짧은 방법 — 변수 2개로 공간 줄이기

## 💡 아이디어

점화식은 사실 직전 두 값만 있으면 계산할 수 있습니다.

```text
dp[i - 1]
dp[i - 2]
```

따라서 DP 배열 전체를 만들 필요가 없습니다.

변수 두 개만 사용합니다.

```text
prev2 = dp[i - 2]
prev1 = dp[i - 1]
```

---

## 💻 C# 코드

```csharp
using System;

public class Solution
{
    public int solution(int[] money)
    {
        int n = money.Length;

        return Math.Max(
            RobLine(money, 0, n - 2),
            RobLine(money, 1, n - 1)
        );
    }

    private int RobLine(int[] money, int start, int end)
    {
        int prev2 = 0;
        int prev1 = 0;

        for (int i = start; i <= end; i++)
        {
            int current = Math.Max(prev1, prev2 + money[i]);

            prev2 = prev1;
            prev1 = current;
        }

        return prev1;
    }
}
```

---

## ✨ 코드 의미

### 1. 이전 두 결과만 저장

```csharp
int prev2 = 0;
int prev1 = 0;
```

처음에는 아직 아무 집도 보지 않았으므로 둘 다 0입니다.

---

### 2. 현재 집까지의 최댓값 계산

```csharp
int current = Math.Max(prev1, prev2 + money[i]);
```

의미는 다음과 같습니다.

```text
prev1              → 현재 집을 털지 않음
prev2 + money[i]   → 현재 집을 털음
```

둘 중 큰 값을 선택합니다.

---

### 3. 값 밀어주기

```csharp
prev2 = prev1;
prev1 = current;
```

다음 집으로 넘어가기 위해 이전 값을 갱신합니다.

---

## ⚠️ LINQ를 쓰지 않은 이유

이 문제는 이전 상태를 계속 갱신하는 DP 문제입니다.

LINQ로 억지로 작성하면 오히려 상태 변화가 잘 보이지 않습니다.

따라서 반복문으로 쓰는 편이 더 안전하고 명확합니다. 🌱

---

## ⏱️ 시간 복잡도

두 범위를 각각 한 번씩 순회합니다.

```text
O(n)
```

입니다.

---

## 📦 공간 복잡도

변수만 사용합니다.

```text
O(1)
```

입니다.

# ❌ 잘못된 접근: 항상 큰 돈이 있는 집부터 고르기

예를 들어 돈이 많은 집부터 고르는 식으로 접근하면 실패할 수 있습니다.

```text
[10, 1, 1, 10]
```

원형 구조에서는 첫 번째 집과 마지막 집을 동시에 털 수 없습니다.

단순히 큰 값만 고르면 인접 조건을 놓칠 수 있습니다. 🚨

그래서 DP로 각 위치까지의 최선 선택을 누적해야 합니다.

# 🏆 추천 풀이

처음 공부할 때는 **풀이 1번 배열 DP**를 추천합니다.

이유는 다음과 같습니다.

```text
1. dp[i]의 의미가 눈에 잘 보입니다.
2. 현재 집을 털지/말지 선택하는 구조를 이해하기 쉽습니다.
3. 원형 문제를 두 개의 일렬 문제로 바꾸는 흐름이 명확합니다.
```

프로그래머스 제출용으로는 **풀이 2번 변수 2개 DP**가 좋습니다.

집 개수가 최대 1,000,000이라 공간을 아끼는 방식이 더 안정적입니다. ⚡

# ✅ 최종 정리

이 문제의 핵심은 다음입니다.

```text
원형이라 첫 번째 집과 마지막 집을 동시에 털 수 없다.
```

그래서 두 경우로 나눕니다.

```text
1. 첫 번째 집을 포함하고 마지막 집을 제외
2. 첫 번째 집을 제외하고 마지막 집을 포함
```

각 경우는 일렬 집 털기 DP로 풉니다.

점화식은 다음입니다.

```text
dp[i] = max(dp[i - 1], dp[i - 2] + money[i])
```

🏠 “도둑질”은 원형 DP를 일렬 DP 두 번으로 바꾸는 대표 문제입니다.
