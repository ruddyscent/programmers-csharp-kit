# 🏫 등굣길 — C# 학습자료

## 📌 1. 문제 핵심

집에서 학교까지 가는 길이 격자 모양으로 주어집니다.

집은 왼쪽 위입니다.

```text
(1, 1)
```

학교는 오른쪽 아래입니다.

```text
(m, n)
```

이동은 두 방향만 가능합니다.

```text
오른쪽 →
아래쪽 ↓
```

중간에 물에 잠긴 칸, 즉 `puddles`는 지나갈 수 없습니다. 🌧️

목표는 다음입니다.

```text
집에서 학교까지 가는 최단 경로의 개수 구하기
```

단, 정답이 커질 수 있으므로:

```text
1,000,000,007로 나눈 나머지
```

를 반환해야 합니다.

---

# 🔍 예제 이해하기

```text
m = 4
n = 3
puddles = [[2, 2]]
```

격자는 가로 4칸, 세로 3칸입니다.

```text
집: (1, 1)
학교: (4, 3)
물웅덩이: (2, 2)
```

물웅덩이를 피해서 오른쪽과 아래쪽으로만 이동해야 합니다.

가능한 최단 경로는 총:

```text
4개
```

입니다. ✅

---

# 🧠 핵심 아이디어

어떤 칸에 도착하는 방법은 어디서 올 수 있을까요?

오른쪽과 아래쪽으로만 이동할 수 있으므로, 현재 칸에는 다음 두 곳에서만 올 수 있습니다.

```text
위쪽 칸
왼쪽 칸
```

따라서 현재 칸까지 오는 경로 수는:

```text
현재 칸 경로 수 = 위쪽 칸 경로 수 + 왼쪽 칸 경로 수
```

입니다.

이것이 DP의 핵심입니다. 🎯

---

# 🧩 DP란?

DP는 Dynamic Programming, 즉 동적 계획법입니다.

쉽게 말하면:

```text
작은 위치까지 오는 방법 수를 저장하고,
그 값을 이용해 다음 위치의 방법 수를 계산하는 방식
```

입니다.

이 문제에서는:

```text
dp[y, x] = (x, y) 칸까지 오는 경로의 개수
```

로 생각하면 됩니다.

---

# ⚠️ 좌표 주의

문제에서 좌표는 다음처럼 주어집니다.

```text
(x, y)
```

즉:

```text
x → 가로 위치
y → 세로 위치
```

하지만 C# 2차원 배열은 보통:

```text
dp[row, col]
```

형태입니다.

따라서:

```text
dp[y, x]
```

처럼 사용하는 것이 자연스럽습니다.

예를 들어 물웅덩이 좌표가:

```text
[2, 2]
```

라면:

```csharp
blocked[2, 2] = true;
```

입니다.

---

# 🧠 풀이 1. 이해하기 쉬운 방법 — 2차원 DP 배열

## 💡 아이디어

1. 물웅덩이 위치를 `blocked` 배열에 표시합니다.
2. 시작점 `(1, 1)`의 경로 수를 1로 둡니다.
3. 왼쪽 위부터 오른쪽 아래까지 순서대로 칸을 확인합니다.
4. 물웅덩이 칸이면 경로 수를 0으로 둡니다.
5. 나머지 칸은 위쪽과 왼쪽의 경로 수를 더합니다.
6. 학교 위치의 값을 반환합니다.

---

## 💻 C# 코드

```csharp
using System;

public class Solution
{
    public int solution(int m, int n, int[,] puddles)
    {
        const int MOD = 1000000007;

        bool[,] blocked = new bool[n + 1, m + 1];

        for (int i = 0; i < puddles.GetLength(0); i++)
        {
            int x = puddles[i, 0];
            int y = puddles[i, 1];

            blocked[y, x] = true;
        }

        int[,] dp = new int[n + 1, m + 1];

        dp[1, 1] = 1;

        for (int y = 1; y <= n; y++)
        {
            for (int x = 1; x <= m; x++)
            {
                if (x == 1 && y == 1)
                {
                    continue;
                }

                if (blocked[y, x])
                {
                    dp[y, x] = 0;
                    continue;
                }

                dp[y, x] = (dp[y - 1, x] + dp[y, x - 1]) % MOD;
            }
        }

        return dp[n, m];
    }
}
```

---

## 🔎 코드 해설

### 1. 나머지 상수

```csharp
const int MOD = 1000000007;
```

문제에서 경로 수를 이 값으로 나눈 나머지를 반환하라고 했습니다.

경로 수가 매우 커질 수 있으므로 계산할 때마다 나머지를 적용합니다.

---

### 2. 물웅덩이 배열 만들기

```csharp
bool[,] blocked = new bool[n + 1, m + 1];
```

좌표를 1부터 사용하기 위해 배열 크기를 `n + 1`, `m + 1`로 만듭니다.

---

### 3. 물웅덩이 표시

```csharp
int x = puddles[i, 0];
int y = puddles[i, 1];

blocked[y, x] = true;
```

문제의 좌표는 `[x, y]`입니다.

C# 배열은 `[y, x]`처럼 행과 열로 접근합니다. ⚠️

---

### 4. 시작점 초기화

```csharp
dp[1, 1] = 1;
```

집에서 집으로 가는 방법은 1가지입니다.

즉, 시작점입니다. 🏠

---

### 5. 시작점은 건너뛰기

```csharp
if (x == 1 && y == 1)
{
    continue;
}
```

시작점 값을 이미 1로 설정했으므로 다시 계산하지 않습니다.

---

### 6. 물웅덩이는 경로 수 0

```csharp
if (blocked[y, x])
{
    dp[y, x] = 0;
    continue;
}
```

물웅덩이는 지나갈 수 없으므로 해당 칸까지 오는 경로 수는 0입니다. 🌧️

---

### 7. 위쪽과 왼쪽에서 오는 경우 더하기

```csharp
dp[y, x] = (dp[y - 1, x] + dp[y, x - 1]) % MOD;
```

현재 칸은 위쪽 또는 왼쪽에서 올 수 있습니다.

그래서 두 값을 더합니다.

---

## ⏱️ 시간 복잡도

격자의 모든 칸을 한 번씩 확인합니다.

```text
O(n × m)
```

입니다.

---

## 📦 공간 복잡도

`blocked` 배열과 `dp` 배열을 사용합니다.

```text
O(n × m)
```

입니다.

# 🚀 풀이 2. 코드가 짧은 방법 — dp 배열 하나만 사용하기

## 💡 아이디어

물웅덩이 정보를 별도 `blocked` 배열로 만들지 않고, `dp` 배열에 `-1`로 표시할 수 있습니다.

```text
-1 → 물웅덩이
0 이상 → 경로 수
```

그다음 계산할 때 물웅덩이면 0으로 바꿔서 지나갈 수 없게 처리합니다.

---

## 💻 C# 코드

```csharp
using System;

public class Solution
{
    public int solution(int m, int n, int[,] puddles)
    {
        const int MOD = 1000000007;

        int[,] dp = new int[n + 1, m + 1];

        for (int i = 0; i < puddles.GetLength(0); i++)
        {
            dp[puddles[i, 1], puddles[i, 0]] = -1;
        }

        dp[1, 1] = 1;

        for (int y = 1; y <= n; y++)
        {
            for (int x = 1; x <= m; x++)
            {
                if (x == 1 && y == 1)
                {
                    continue;
                }

                if (dp[y, x] == -1)
                {
                    dp[y, x] = 0;
                    continue;
                }

                dp[y, x] = (dp[y - 1, x] + dp[y, x - 1]) % MOD;
            }
        }

        return dp[n, m];
    }
}
```

---

## ✨ 코드 의미

### 1. 물웅덩이를 -1로 표시

```csharp
dp[puddles[i, 1], puddles[i, 0]] = -1;
```

물웅덩이 좌표는 `[x, y]`이므로 배열에는 `[y, x]`로 넣습니다.

---

### 2. 계산 중 물웅덩이 만나면 0으로 변경

```csharp
if (dp[y, x] == -1)
{
    dp[y, x] = 0;
    continue;
}
```

물웅덩이는 지나갈 수 없으므로 경로 수를 0으로 만듭니다.

---

### 3. 나머지는 동일

```csharp
dp[y, x] = (dp[y - 1, x] + dp[y, x - 1]) % MOD;
```

위쪽과 왼쪽 경로 수를 더합니다.

---

## ⚠️ LINQ를 쓰지 않은 이유

이 문제는 격자의 각 칸을 순서대로 누적 계산해야 합니다.

```text
위쪽 값과 왼쪽 값이 먼저 계산되어 있어야 함
```

이런 문제는 LINQ보다 반복문이 더 명확하고 안전합니다. 🌱

---

## ⏱️ 시간 복잡도

```text
O(n × m)
```

입니다.

---

## 📦 공간 복잡도

`dp` 배열 하나만 사용합니다.

```text
O(n × m)
```

입니다.

풀이 1보다 배열 하나를 덜 사용합니다. ⚡

# 💡 참고: 1차원 DP도 가능할까?

가능합니다.

`dp[x]`를 현재 행에서 x 위치까지 오는 방법 수로 관리하면 됩니다.

하지만 처음 공부할 때는 2차원 DP가 훨씬 이해하기 쉽습니다.

이 문제는 좌표 실수가 자주 나므로, 먼저 2차원 DP로 확실히 이해하는 것을 추천합니다. 😊

| 풀이 2 | `dp`에 물웅덩이 표시 | 코드가 짧고 배열이 하나 적다 🚀 | O(n × m) | O(n × m) |

# 🏆 추천 풀이

처음 공부할 때는 **풀이 1번**을 추천합니다.

이유는 다음과 같습니다.

```text
1. 물웅덩이 배열과 경로 수 배열이 분리되어 이해하기 쉽습니다.
2. 좌표 변환을 명확히 볼 수 있습니다.
3. DP의 의미가 잘 드러납니다.
```

프로그래머스 제출용으로는 **풀이 2번**도 좋습니다.

# ✅ 최종 정리

이 문제의 핵심은 다음입니다.

```text
현재 칸까지 오는 방법 수 = 위쪽 칸 + 왼쪽 칸
```

단, 물웅덩이는 지나갈 수 없으므로:

```text
물웅덩이 칸의 경로 수 = 0
```

입니다.

풀이 흐름은 다음과 같습니다.

```text
1. 물웅덩이 위치를 표시한다.
2. 시작점 dp[1,1]을 1로 둔다.
3. 왼쪽 위부터 오른쪽 아래까지 순서대로 계산한다.
4. 물웅덩이는 0으로 처리한다.
5. 학교 위치 dp[n,m]을 반환한다.
```

🏫 “등굣길”은 격자 DP와 좌표 처리를 함께 연습하기 좋은 대표 문제입니다.
