# 🔢 같은 숫자는 싫어 — C# 학습자료

## 📌 1. 문제 핵심

배열 `arr`가 주어집니다.

배열 안에는 `0`부터 `9`까지의 숫자가 들어 있습니다.

해야 할 일은 다음입니다.

```text
연속해서 같은 숫자가 나오면 하나만 남긴다.
```

예를 들어:

```text
arr = [1, 1, 3, 3, 0, 1, 1]
```

이라면 연속 중복을 제거해서:

```text
[1, 3, 0, 1]
```

을 반환해야 합니다. ✅

## ⚠️ 주의할 점

이 문제는 **전체 중복 제거**가 아닙니다.

예를 들어:

```text
[1, 1, 3, 3, 0, 1, 1]
```

에서 `1`은 앞에도 나오고 뒤에도 나옵니다.

하지만 뒤쪽 `1`은 앞의 `0` 다음에 새롭게 나온 숫자이므로 남겨야 합니다.

정답은:

```text
[1, 3, 0, 1]
```

입니다.

만약 전체 중복 제거를 해버리면:

```text
[1, 3, 0]
```

이 되어 틀립니다. 🚨

# 🧠 핵심 아이디어

현재 숫자가 바로 앞에 넣은 숫자와 다를 때만 결과에 추가합니다.

즉:

```text
현재 숫자 != 결과의 마지막 숫자
```

이면 추가합니다.

예를 들어:

```text
[1, 1, 3, 3, 0, 1, 1]
```

을 왼쪽부터 보면:

```text
1 → 처음 숫자이므로 추가
1 → 바로 앞 숫자와 같으므로 무시
3 → 앞 숫자 1과 다르므로 추가
3 → 앞 숫자 3과 같으므로 무시
0 → 앞 숫자 3과 다르므로 추가
1 → 앞 숫자 0과 다르므로 추가
1 → 앞 숫자 1과 같으므로 무시
```

결과:

```text
[1, 3, 0, 1]
```

🎯 핵심은 “바로 앞 숫자와만 비교한다”입니다.

# 🔍 예제 1

```text
arr = [1, 1, 3, 3, 0, 1, 1]
```

처리 과정:

| 현재 숫자 | 결과 배열 | 설명 |
|---:|---|---|
| 1 | [1] | 처음 숫자라 추가 😊 |
| 1 | [1] | 앞 숫자와 같아서 무시 |
| 3 | [1, 3] | 앞 숫자와 달라서 추가 |
| 3 | [1, 3] | 앞 숫자와 같아서 무시 |
| 0 | [1, 3, 0] | 앞 숫자와 달라서 추가 |
| 1 | [1, 3, 0, 1] | 앞 숫자와 달라서 추가 |
| 1 | [1, 3, 0, 1] | 앞 숫자와 같아서 무시 |

정답:

```text
[1, 3, 0, 1]
```

# 🔍 예제 2

```text
arr = [4, 4, 4, 3, 3]
```

처리 과정:

```text
4 → 추가
4 → 무시
4 → 무시
3 → 추가
3 → 무시
```

정답:

```text
[4, 3]
```

✅

# 🧠 풀이 1. 이해하기 쉬운 방법 — List에 직접 추가하기

## 💡 아이디어

결과를 담을 `List<int>`를 만듭니다.

그리고 `arr`를 처음부터 끝까지 보면서:

```text
결과가 비어 있으면 추가
결과의 마지막 숫자와 현재 숫자가 다르면 추가
같으면 무시
```

합니다.

## 💻 C# 코드

```csharp
using System.Collections.Generic;

public class Solution
{
    public int[] solution(int[] arr)
    {
        List<int> answer = new List<int>();

        foreach (int number in arr)
        {
            if (answer.Count == 0 || answer[answer.Count - 1] != number)
                answer.Add(number);
        }

        return answer.ToArray();
    }
}
```

## 🔎 코드 해설

### 1. 결과를 담을 List 만들기

```csharp
List<int> answer = new List<int>();
```

결과 배열의 크기를 처음부터 알 수 없으므로 `List<int>`를 사용합니다.

`List<int>`는 원소를 하나씩 추가하기 좋습니다. 📦

### 2. 배열을 왼쪽부터 확인하기

```csharp
foreach (int number in arr)
```

배열의 숫자를 순서대로 하나씩 확인합니다.

순서를 유지해야 하므로 왼쪽부터 처리합니다. 👀

### 3. 처음 숫자는 무조건 추가

```csharp
answer.Count == 0
```

결과 배열이 비어 있다면 아직 비교할 숫자가 없습니다.

따라서 첫 숫자는 무조건 추가합니다.

### 4. 마지막 숫자와 다르면 추가

```csharp
answer[answer.Count - 1] != number
```

결과 배열의 마지막 숫자와 현재 숫자가 다르면 추가합니다.

예를 들어 결과가:

```text
[1, 3]
```

이고 현재 숫자가 `0`이면 다르므로 추가합니다.

```text
[1, 3, 0]
```

### 5. 배열로 변환해서 반환

```csharp
return answer.ToArray();
```

프로그래머스의 반환 타입은 `int[]`입니다.

따라서 `List<int>`를 배열로 바꿔서 반환합니다. ✅

## ⏱️ 시간 복잡도

배열을 한 번만 순회합니다.

```text
O(n)
```

`n`은 `arr.Length`입니다.

## 📦 공간 복잡도

결과를 저장하는 List를 사용합니다.

최악의 경우 연속 중복이 하나도 없으면 모든 숫자를 저장합니다.

```text
O(n)
```

입니다.

# 🚀 풀이 2. 짧은 코드 버전 — LINQ Where 사용하기

## 💡 아이디어

현재 위치 `i`의 숫자를 남길지 판단합니다.

남기는 조건은 다음과 같습니다.

```text
i == 0 이거나 arr[i] != arr[i - 1]
```

즉:

```text
첫 번째 숫자는 항상 남긴다.
그 뒤 숫자는 바로 앞 숫자와 다를 때만 남긴다.
```

## 💻 C# 코드

```csharp
using System.Linq;

public class Solution
{
    public int[] solution(int[] arr)
    {
        return arr
            .Where((number, index) => index == 0 || number != arr[index - 1])
            .ToArray();
    }
}
```

## ✨ 코드 의미

### 1. `Where`로 남길 숫자 고르기

```csharp
.Where((number, index) => index == 0 || number != arr[index - 1])
```

`Where`는 조건을 만족하는 원소만 남깁니다.

여기서는 두 가지 경우를 남깁니다.

```text
첫 번째 원소
또는
바로 앞 원소와 다른 원소
```

### 2. 첫 번째 원소 처리

```csharp
index == 0
```

첫 번째 숫자는 앞 숫자가 없으므로 무조건 남깁니다.

### 3. 앞 숫자와 비교

```csharp
number != arr[index - 1]
```

현재 숫자가 바로 앞 숫자와 다르면 남깁니다.

같으면 연속 중복이므로 제거합니다. 🧹

### 4. 배열로 변환

```csharp
.ToArray()
```

프로그래머스 반환 타입이 `int[]`이므로 배열로 변환합니다.

## ⚠️ 프로그래머스에서 실행할 때 주의

LINQ를 사용하므로 반드시 아래 네임스페이스가 필요합니다.

```csharp
using System.Linq;
```

프로그래머스 C# 환경에서 실행 가능합니다. ✅

## ⏱️ 시간 복잡도

`Where`가 배열을 한 번 순회합니다.

```text
O(n)
```

입니다.

## 📦 공간 복잡도

결과 배열을 만듭니다.

```text
O(n)
```

입니다.

LINQ 내부 반복자도 사용하지만 전체적으로는 결과 배열 크기가 가장 큽니다.

# ⚖️ 두 풀이 비교

| 풀이 | 핵심 방법 | 장점 | 시간 복잡도 | 공간 복잡도 |
|---|---|---|---:|---:|
| 풀이 1 | `List<int>`에 직접 추가 | 원리가 잘 보인다 😊 | O(n) | O(n) |
| 풀이 2 | LINQ `Where` | 코드가 짧다 🚀 | O(n) | O(n) |

# 🏆 추천 풀이

처음 공부할 때는 **풀이 1번 List 방식**을 추천합니다.

이유는 다음과 같습니다.

```text
1. 연속 중복을 제거하는 흐름이 잘 보입니다.
2. 결과의 마지막 값과 비교한다는 핵심이 명확합니다.
3. 디버깅하기 쉽습니다.
```

코드를 짧게 쓰고 싶다면 **풀이 2번 LINQ 방식**도 좋습니다.

프로그래머스 C#에서 실행할 수 있으며, `using System.Linq;`만 잊지 않으면 됩니다. ⚡

# ✅ 최종 정리

이 문제는 전체 중복 제거가 아닙니다.

```text
연속으로 붙어 있는 같은 숫자만 제거한다.
```

핵심 조건은 다음입니다.

```text
첫 숫자는 남긴다.
그다음부터는 바로 앞 숫자와 다를 때만 남긴다.
```

즉:

```text
arr[i] != arr[i - 1]
```

일 때만 결과에 넣으면 됩니다.

🔢 “같은 숫자는 싫어”는 이전 값과 현재 값을 비교하는 간단한 1회 순회 문제입니다.
