# ✅ 올바른 괄호 — C# 학습자료

## 📌 1. 문제 핵심

문자열 `s`는 `'('`와 `')'`로만 이루어져 있습니다.

이 문자열이 **올바른 괄호 문자열**인지 확인해야 합니다.

올바른 괄호란 다음 조건을 만족하는 문자열입니다.

```text
'('로 열렸으면 반드시 나중에 ')'로 닫혀야 한다.
```

예를 들어:

```text
"()()"    → 올바름
"(())()"  → 올바름
")()("    → 올바르지 않음
"(()("    → 올바르지 않음
```

---

## 🧠 핵심 아이디어

괄호를 왼쪽부터 하나씩 읽습니다.

`(`가 나오면 아직 닫히지 않은 괄호가 하나 늘어납니다.

`)`가 나오면 열려 있던 괄호 하나를 닫습니다.

즉:

```text
'(' → +1
')' → -1
```

로 생각할 수 있습니다. 🎯

---

## ⚠️ 올바르지 않은 경우

### 1. 중간에 닫는 괄호가 더 많아지는 경우

예를 들어:

```text
")()("
```

처음부터 `)`가 나옵니다.

닫으려면 먼저 열린 괄호 `(`가 있어야 하는데 없습니다.

이 경우는 바로 실패입니다. ❌

---

### 2. 끝났는데 열린 괄호가 남은 경우

예를 들어:

```text
"(()("
```

왼쪽부터 확인하면 열린 괄호가 남습니다.

즉, 닫히지 않은 `(`가 있습니다.

이 경우도 실패입니다. ❌

---

# 🧠 풀이 1. 이해하기 쉬운 방법 — Stack 사용하기

## 💡 아이디어

괄호 문제는 보통 Stack으로 풉니다.

`(`가 나오면 Stack에 넣습니다.

`)`가 나오면 Stack에서 하나 꺼냅니다.

단, Stack이 비어 있는데 `)`가 나오면 잘못된 괄호입니다.

마지막에 Stack이 비어 있으면 올바른 괄호입니다. 📦

---

## 💻 C# 코드

```csharp
using System;
using System.Collections.Generic;

public class Solution
{
    public bool solution(string s)
    {
        Stack<char> stack = new Stack<char>();

        foreach (char ch in s)
        {
            if (ch == '(')
            {
                stack.Push(ch);
            }
            else
            {
                if (stack.Count == 0)
                {
                    return false;
                }

                stack.Pop();
            }
        }

        return stack.Count == 0;
    }
}
```

---

## 🔎 코드 해설

### 1. Stack 만들기

```csharp
Stack<char> stack = new Stack<char>();
```

아직 닫히지 않은 `(`를 저장합니다.

---

### 2. 여는 괄호 처리

```csharp
if (ch == '(')
{
    stack.Push(ch);
}
```

`(`가 나오면 나중에 닫혀야 하므로 Stack에 넣습니다. 🧺

---

### 3. 닫는 괄호 처리

```csharp
else
{
    if (stack.Count == 0)
    {
        return false;
    }

    stack.Pop();
}
```

`)`가 나오면 앞에 나온 `(` 하나와 짝을 지어야 합니다.

Stack이 비어 있다면 짝이 될 `(`가 없다는 뜻입니다.

따라서 바로 `false`를 반환합니다. 🚨

---

### 4. 마지막 확인

```csharp
return stack.Count == 0;
```

문자열을 끝까지 확인했는데 Stack이 비어 있으면 모든 괄호가 잘 닫힌 것입니다.

Stack에 `(`가 남아 있다면 닫히지 않은 괄호가 있다는 뜻입니다.

---

## ⏱️ 시간 복잡도

문자열을 한 번만 순회합니다.

```text
O(n)
```

`n`은 문자열 길이입니다.

---

## 📦 공간 복잡도

최악의 경우 모든 문자가 `(`일 수 있습니다.

이때 Stack에 모든 문자가 들어갑니다.

```text
O(n)
```

입니다.

# 🚀 풀이 2. 코드가 짧은 방법 — 카운터 하나로 풀기

## 💡 아이디어

이 문제는 괄호 종류가 하나뿐입니다.

따라서 Stack을 쓰지 않아도 됩니다.

열린 괄호 개수만 세면 됩니다.

```text
'(' → count++
')' → count--
```

중간에 `count`가 음수가 되면 잘못된 괄호입니다.

마지막에 `count == 0`이면 올바른 괄호입니다. ✅

---

## 💻 C# 코드

```csharp
using System;

public class Solution
{
    public bool solution(string s)
    {
        int count = 0;

        foreach (char ch in s)
        {
            count += ch == '(' ? 1 : -1;

            if (count < 0)
            {
                return false;
            }
        }

        return count == 0;
    }
}
```

---

## ✨ 코드 의미

### 1. 열린 괄호 수 세기

```csharp
int count = 0;
```

현재까지 닫히지 않은 `(`의 개수를 의미합니다.

---

### 2. 괄호에 따라 증가 또는 감소

```csharp
count += ch == '(' ? 1 : -1;
```

현재 문자가 `(`이면 1 증가합니다.

현재 문자가 `)`이면 1 감소합니다.

---

### 3. 음수가 되면 실패

```csharp
if (count < 0)
{
    return false;
}
```

`count`가 음수라는 것은 닫는 괄호가 더 많다는 뜻입니다.

예를 들어:

```text
")("
```

처음부터 `)`가 나오면 `count = -1`이 됩니다.

이 경우는 바로 실패입니다. 🚫

---

### 4. 마지막에 0인지 확인

```csharp
return count == 0;
```

끝까지 확인한 뒤 `count`가 0이면 모든 괄호가 짝을 이룬 것입니다.

`count`가 1 이상이면 닫히지 않은 `(`가 남아 있는 것입니다.

---

## ⏱️ 시간 복잡도

문자열을 한 번만 확인합니다.

```text
O(n)
```

---

## 📦 공간 복잡도

정수 변수 하나만 사용합니다.

```text
O(1)
```

---

# ✨ 참고: LINQ로도 가능할까?

LINQ만으로 누적 상태를 검사하려면 중간에 `count < 0`이 되는 순간을 확인해야 합니다.

가능은 하지만, 오히려 코드가 복잡해지고 가독성이 떨어집니다.

이 문제에서는 LINQ보다 단순한 `foreach`가 더 좋습니다. 😊

프로그래머스 제출용으로도 카운터 풀이가 가장 깔끔합니다.

---

# ⚖️ 두 풀이 비교

| 풀이 | 핵심 방법 | 장점 | 시간 복잡도 | 공간 복잡도 |
|---|---|---|---:|---:|
| 풀이 1 | Stack 사용 | 괄호 문제의 기본 원리가 잘 보인다 😊 | O(n) | O(n) |
| 풀이 2 | 카운터 사용 | 코드가 짧고 메모리를 적게 쓴다 🚀 | O(n) | O(1) |

# 🏆 추천 풀이

처음 괄호 문제를 공부한다면 **풀이 1번 Stack 방식**을 먼저 이해하는 것이 좋습니다.

이유는 다음과 같습니다.

```text
1. 괄호 짝 맞추기 문제의 기본 구조입니다.
2. 여러 종류의 괄호 문제로 확장하기 쉽습니다.
3. Stack 자료구조를 연습하기 좋습니다.
```

하지만 이 문제만 놓고 보면 **풀이 2번 카운터 방식**이 더 좋습니다.

```text
괄호 종류가 '('와 ')' 하나뿐이기 때문입니다.
```

그래서 프로그래머스 제출용으로는 카운터 풀이를 추천합니다. ⚡

# ✅ 최종 정리

이 문제의 핵심은 다음입니다.

```text
왼쪽부터 읽으면서 열린 괄호 수를 센다.
```

규칙은 간단합니다.

```text
'('이면 +1
')'이면 -1
중간에 음수가 되면 false
마지막에 0이면 true
```

🥨 “올바른 괄호”는 Stack으로도 풀 수 있지만, 괄호 종류가 하나뿐이라 카운터 하나로 충분합니다.
