# 🛂 입국심사 — C# 학습자료

## 📌 1. 문제 핵심

입국심사를 기다리는 사람이 `n`명 있습니다.

심사관마다 한 사람을 심사하는 데 걸리는 시간이 다릅니다.

우리가 구해야 하는 것은 다음입니다.

```text
모든 사람이 심사를 끝내는 데 필요한 최소 시간
```

입니다. 🎯

예를 들어 심사 시간이 다음과 같다고 해봅시다.

```text
times = [7, 10]
n = 6
```

28분이 지나면:

```text
7분 심사관  → 28 / 7  = 4명
10분 심사관 → 28 / 10 = 2명
```

총 6명을 심사할 수 있습니다.

따라서 답은 `28`입니다. ✅

---

## ⚠️ 직접 시뮬레이션하면 위험한 이유

가장 먼저 떠올릴 수 있는 방법은 사람을 한 명씩 심사대에 배정하는 것입니다.

하지만 `n`은 최대 1,000,000,000명입니다.

```text
한 명씩 처리하면 너무 오래 걸린다.
```

그래서 사람을 직접 배정하지 않습니다.

대신 시간을 정해 놓고 다음 질문을 합니다.

```text
이 시간 안에 n명을 모두 심사할 수 있을까?
```

이 질문에 답하면서 가능한 최소 시간을 찾습니다. 🔍

---

# 🧠 핵심 아이디어: 시간을 이분 탐색하기

어떤 시간 `x`가 주어졌다고 해봅시다.

심사 시간이 `t`분인 심사관은 `x`분 동안 다음 명수를 처리할 수 있습니다.

```text
x / t
```

예를 들어 `x = 28`이고 `times = [7, 10]`이면:

```text
28 / 7  = 4
28 / 10 = 2
```

총 6명입니다.

즉, 특정 시간 안에 처리할 수 있는 사람 수는 다음처럼 계산합니다.

```text
sum(시간 / 각 심사관의 심사 시간)
```

이 값이 `n` 이상이면:

```text
그 시간 안에 모두 심사 가능
```

입니다. ✅

반대로 `n`보다 작으면:

```text
시간이 부족함
```

입니다. 🚫

---

## 🔎 왜 이분 탐색이 가능할까?

시간이 길어질수록 심사할 수 있는 사람 수는 줄지 않습니다.

```text
10분에 가능했다면 11분, 12분도 가능하다.
10분에 불가능했다면 더 짧은 시간도 불가능하다.
```

이렇게 결과가 한 방향으로 바뀌는 구조입니다.

```text
불가능 불가능 불가능 가능 가능 가능
```

따라서 가능한 시간 중 가장 작은 값을 이분 탐색으로 찾을 수 있습니다. 🚀

---

# 🧠 풀이 1. 이해하기 쉬운 방법 — 정렬 후 이분 탐색

## 💡 아이디어

1. 가장 빠른 심사관 시간을 찾기 위해 `times`를 정렬합니다.
2. 최소 시간 `left`는 `1`입니다.
3. 최대 시간 `right`는 가장 빠른 심사관이 혼자 `n`명을 처리하는 시간입니다.
4. 중간 시간 `mid` 안에 몇 명을 심사할 수 있는지 계산합니다.
5. `n`명 이상 가능하면 답 후보로 저장하고 더 짧은 시간을 찾습니다.
6. 부족하면 시간을 늘립니다.

---

## 💻 C# 코드

```csharp
using System;

public class Solution
{
    public long solution(int n, int[] times)
    {
        Array.Sort(times);

        long left = 1;
        long right = (long)times[0] * n;
        long answer = right;

        while (left <= right)
        {
            long mid = (left + right) / 2;
            long passed = 0;

            foreach (int time in times)
            {
                passed += mid / time;

                if (passed >= n)
                    break;
            }

            if (passed >= n)
            {
                answer = mid;
                right = mid - 1;
            }
            else
            {
                left = mid + 1;
            }
        }

        return answer;
    }
}
```

---

## 🔎 코드 해설

### 1. 심사 시간을 정렬

```csharp
Array.Sort(times);
```

정렬하면 가장 빠른 심사관의 시간이 `times[0]`에 옵니다.

이 값을 이용해 최대 시간을 잡습니다.

---

### 2. 탐색 범위 설정

```csharp
long left = 1;
long right = (long)times[0] * n;
```

최소 시간은 1분입니다.

최대 시간은 가장 빠른 심사관이 혼자 모든 사람을 심사하는 경우입니다.

`n`과 `times[i]`가 최대 1,000,000,000까지 커질 수 있으므로 `int`를 넘을 수 있습니다.

그래서 `long`을 사용해야 합니다. ⚠️

---

### 3. mid 시간 안에 처리 가능한 사람 수 계산

```csharp
long passed = 0;

foreach (int time in times)
{
    passed += mid / time;

    if (passed >= n)
        break;
}
```

각 심사관이 `mid`분 동안 처리할 수 있는 사람 수를 더합니다.

이미 `n`명 이상이면 더 계산할 필요가 없으므로 멈춥니다.

이렇게 하면 불필요한 반복을 줄이고, `passed`가 너무 커지는 것도 막을 수 있습니다. ✅

---

### 4. 가능한 시간이면 더 줄이기

```csharp
if (passed >= n)
{
    answer = mid;
    right = mid - 1;
}
```

`mid`분 안에 모두 심사할 수 있다면 `mid`는 답 후보입니다.

하지만 더 짧은 시간도 가능할 수 있으므로 왼쪽 범위를 탐색합니다.

---

### 5. 불가능하면 시간 늘리기

```csharp
else
{
    left = mid + 1;
}
```

`mid`분 안에 `n`명을 처리하지 못했다면 시간이 부족합니다.

따라서 더 긴 시간을 탐색합니다. ⏱️

---

## ⏱️ 시간 복잡도

심사관 수를 `m`, 가능한 최대 시간을 `T`라고 하겠습니다.

정렬에 다음 시간이 걸립니다.

```text
O(m log m)
```

이분 탐색은 약 `log T`번 반복하고, 매번 심사관을 한 번씩 확인합니다.

```text
O(m log T)
```

따라서 전체 시간 복잡도는 다음과 같습니다.

```text
O(m log m + m log T)
```

---

## 📦 공간 복잡도

별도의 큰 자료구조를 만들지 않습니다.

```text
O(1)
```

단, 정렬 내부 구현에 따라 약간의 추가 공간이 사용될 수 있습니다.

# 🚀 풀이 2. 짧은 코드 버전 — lower bound(가능한 가장 작은 값 찾기) 방식

## 💡 아이디어

풀이 1과 같은 이분 탐색입니다.

다만 `answer` 변수를 따로 두지 않고, `left < right` 형태로 범위를 좁힙니다.

반복이 끝나면 `left`가 가능한 최소 시간입니다. ✨

---

## 💻 C# 코드

```csharp
using System.Linq;

public class Solution
{
    public long solution(int n, int[] times)
    {
        long left = 1, right = (long)times.Min() * n;

        while (left < right)
        {
            long mid = (left + right) / 2;
            long count = 0;

            foreach (int time in times)
            {
                count += mid / time;

                if (count >= n)
                    break;
            }

            if (count >= n)
                right = mid;
            else
                left = mid + 1;
        }

        return left;
    }
}
```

---

## ✨ 코드 의미

### 1. 오른쪽 경계 설정

```csharp
long left = 1, right = (long)times.Min() * n;
```

`times.Min()`으로 가장 빠른 심사 시간을 구합니다.

그 심사관이 혼자 `n`명을 처리하는 시간을 오른쪽 경계로 잡습니다.

---

### 2. 가능한 시간은 오른쪽을 줄이기

```csharp
if (count >= n)
    right = mid;
```

`mid`분 안에 모두 심사할 수 있다면 `mid`도 답 후보입니다.

그래서 `mid`를 버리지 않고 `right = mid`로 줄입니다.

---

### 3. 불가능한 시간은 왼쪽을 올리기

```csharp
else
    left = mid + 1;
```

`mid`분이 부족하다면 `mid` 이하의 시간은 모두 불가능합니다.

따라서 `left`를 `mid + 1`로 옮깁니다.

---

## ⚠️ 프로그래머스에서 실행할 때 주의

`times.Min()`을 사용하므로 다음 네임스페이스가 필요합니다.

```csharp
using System.Linq;
```

반환 타입도 반드시 `long`이어야 합니다.

```csharp
public long solution(int n, int[] times)
```

---

## ⏱️ 시간 복잡도

심사관 수를 `m`, 가능한 최대 시간을 `T`라고 하겠습니다.

`Min()`으로 최소 심사 시간을 찾는 데 `O(m)`이 걸립니다.

이분 탐색은 `log T`번 반복하고, 매번 심사관을 확인합니다.

```text
O(m log T)
```

입니다.

---

## 📦 공간 복잡도

추가 배열이나 컬렉션을 만들지 않습니다.

```text
O(1)
```

입니다.

# ⚖️ 두 풀이 비교

| 풀이 | 핵심 방법 | 장점 | 시간 복잡도 | 공간 복잡도 |
|---|---|---|---:|---:|
| 풀이 1 | 정렬 + 이분 탐색 | 흐름이 명확하다 😊 | O(m log m + m log T) | O(1) |
| 풀이 2 | lower bound(가능한 가장 작은 값 찾기) 이분 탐색 | 코드가 더 짧다 🚀 | O(m log T) | O(1) |

# 🏆 추천 풀이

처음 공부할 때는 **풀이 1번**을 추천합니다.

이유는 다음과 같습니다.

```text
1. left, right, answer의 의미가 잘 보입니다.
2. 가능한 시간을 찾으면 답 후보를 저장하는 흐름이 직관적입니다.
3. 이분 탐색의 기본 구조를 익히기 좋습니다.
```

코딩 테스트 제출용으로 코드 길이를 우선한다면 **풀이 2번**이 더 좋습니다.

다만 `left < right` 방식이 익숙하지 않다면 풀이 1번으로 먼저 이해하는 것이 안전합니다. 🌱

# ✅ 최종 정리

이 문제는 사람을 직접 배정하는 문제가 아닙니다.

핵심은 다음입니다.

```text
특정 시간 안에 몇 명을 심사할 수 있는지 계산한다.
```

그리고 그 시간이 충분한지 확인합니다.

```text
처리 가능 인원 >= n
```

이면 가능한 시간입니다.

이분 탐색으로 가능한 시간 중 가장 작은 값을 찾으면 됩니다.

🛂 “입국심사”는 정답의 범위를 탐색하는 대표적인 이분 탐색 문제입니다.
