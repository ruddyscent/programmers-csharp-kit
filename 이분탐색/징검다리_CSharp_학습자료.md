# 🌉 징검다리 — C# 학습자료

## 📌 1. 문제 핵심

출발 지점은 `0`, 도착 지점은 `distance`입니다.

그 사이에 여러 바위가 있고, 바위 `n`개를 제거할 수 있습니다.

목표는 다음입니다.

```text
바위를 n개 제거한 뒤,
인접한 지점 사이 거리의 최솟값을 최대한 크게 만들기
```

입니다. 🎯

예를 들어:

```text
distance = 25
rocks = [2, 14, 11, 21, 17]
n = 2
```

바위 `2`, `14`를 제거하면 남은 지점은 다음과 같습니다.

```text
0, 11, 17, 21, 25
```

인접 거리들은:

```text
11, 6, 4, 4
```

이고, 이 중 최솟값은 `4`입니다.

가능한 경우들 중 이 최솟값을 가장 크게 만들면 답은 `4`입니다. ✅

---

# ⚠️ 모든 제거 조합을 보면 안 되는 이유

바위는 최대 50,000개입니다.

이 중에서 `n`개를 제거하는 모든 조합을 직접 확인하면 경우의 수가 너무 큽니다.

```text
어떤 바위를 제거할지 직접 고르는 방식은 불가능
```

그래서 관점을 바꿔야 합니다.

```text
최소 거리를 x 이상으로 만들 수 있을까?
```

이 질문에 답하면서 가능한 가장 큰 `x`를 찾습니다. 🔍

---

# 🧠 핵심 아이디어: 최소 거리 값을 이분 탐색하기

정답은 “인접 거리의 최솟값”입니다.

이 값은 최소 1부터 최대 `distance`까지 가능합니다.

어떤 값 `x`를 정해봅시다.

```text
모든 인접 거리를 x 이상으로 만들 수 있는가?
```

이를 확인하려면 바위를 왼쪽부터 보면서 판단합니다.

직전으로 남긴 지점과 현재 바위 사이의 거리가 `x`보다 작다면:

```text
현재 바위를 제거해야 한다.
```

왜냐하면 현재 바위를 남기면 인접 거리 조건을 만족하지 못하기 때문입니다.

반대로 거리가 `x` 이상이면:

```text
현재 바위를 남길 수 있다.
```

이렇게 필요한 제거 횟수를 세었을 때 `n`개 이하라면 `x`는 가능한 최소 거리입니다. ✅

---

## 🔎 왜 이분 탐색이 가능할까?

최소 거리를 크게 만들수록 조건은 어려워집니다.

```text
최소 거리 3 가능 → 최소 거리 2도 가능
최소 거리 5 불가능 → 최소 거리 6도 불가능
```

즉, 가능 여부가 한 방향으로 바뀝니다.

```text
가능 가능 가능 불가능 불가능
```

따라서 가능한 값 중 가장 큰 값을 이분 탐색으로 찾을 수 있습니다. 🚀

---

# 🧠 풀이 1. 이해하기 쉬운 방법 — 정렬 후 이분 탐색

## 💡 아이디어

1. 바위 위치를 오름차순 정렬합니다.
2. 최소 거리 후보를 `left = 1`, `right = distance`로 둡니다.
3. 중간값 `mid`를 최소 거리로 만들 수 있는지 확인합니다.
4. 만들 수 있다면 답 후보로 저장하고 더 큰 거리를 시도합니다.
5. 만들 수 없다면 더 작은 거리를 시도합니다.

---

## 💻 C# 코드

```csharp
using System;

public class Solution
{
    public int solution(int distance, int[] rocks, int n)
    {
        Array.Sort(rocks);

        int left = 1;
        int right = distance;
        int answer = 0;

        while (left <= right)
        {
            int mid = (left + right) / 2;
            int removed = 0;
            int previous = 0;

            foreach (int rock in rocks)
            {
                if (rock - previous < mid)
                {
                    removed++;
                }
                else
                {
                    previous = rock;
                }
            }

            if (distance - previous < mid)
                removed++;

            if (removed <= n)
            {
                answer = mid;
                left = mid + 1;
            }
            else
            {
                right = mid - 1;
            }
        }

        return answer;
    }
}
```

---

## 🔎 코드 해설

### 1. 바위 정렬

```csharp
Array.Sort(rocks);
```

바위를 왼쪽에서 오른쪽으로 확인해야 합니다.

그래야 직전에 남긴 지점과 현재 바위 사이의 거리를 계산할 수 있습니다.

---

### 2. 탐색 범위 설정

```csharp
int left = 1;
int right = distance;
```

우리가 찾는 값은 인접 거리의 최솟값입니다.

최소 후보는 `1`, 최대 후보는 전체 거리 `distance`입니다.

---

### 3. 제거해야 하는 바위 수 세기

```csharp
if (rock - previous < mid)
{
    removed++;
}
else
{
    previous = rock;
}
```

`previous`는 마지막으로 남긴 지점입니다.

현재 바위와 `previous` 사이의 거리가 `mid`보다 작다면 현재 바위를 남길 수 없습니다.

그래서 제거 개수를 증가시킵니다. 🪨

거리가 충분하면 현재 바위를 남기고 `previous`를 현재 바위로 바꿉니다.

---

### 4. 마지막 바위와 도착점 사이 확인

```csharp
if (distance - previous < mid)
    removed++;
```

도착점도 마지막 지점으로 생각해야 합니다.

마지막으로 남긴 바위와 도착점 사이가 `mid`보다 작다면, 마지막 구간을 맞추기 위해 바위 하나를 더 제거해야 한다고 볼 수 있습니다.

---

### 5. 가능한 거리면 더 크게 시도

```csharp
if (removed <= n)
{
    answer = mid;
    left = mid + 1;
}
```

`mid`를 만들기 위해 제거해야 하는 바위 수가 `n`개 이하라면 가능합니다.

정확히 `n`개를 제거해야 하더라도 괜찮습니다.

남는 제거 횟수는 조건을 망치지 않는 바위를 추가로 제거하는 데 쓸 수 있습니다.

그래서 더 큰 최소 거리를 시도합니다. 🚀

---

### 6. 불가능하면 거리 줄이기

```csharp
else
{
    right = mid - 1;
}
```

제거해야 하는 바위 수가 `n`개보다 많다면 `mid`는 너무 큰 값입니다.

최소 거리 후보를 줄여야 합니다.

---

## ⏱️ 시간 복잡도

바위 개수를 `r`, 전체 거리를 `D`라고 하겠습니다.

정렬에 다음 시간이 걸립니다.

```text
O(r log r)
```

이분 탐색은 `log D`번 반복하고, 매번 바위를 한 번 순회합니다.

```text
O(r log D)
```

따라서 전체 시간 복잡도는:

```text
O(r log r + r log D)
```

입니다.

---

## 📦 공간 복잡도

별도의 큰 자료구조를 만들지 않습니다.

```text
O(1)
```

단, 정렬 내부 구현에 따라 약간의 추가 공간이 사용될 수 있습니다.

# 🚀 풀이 2. 짧은 코드 버전 — upper bound(가능한 가장 큰 값 찾기) 방식

## 💡 아이디어

풀이 1과 같은 이분 탐색입니다.

다만 `answer` 변수를 따로 두지 않고, `left < right` 형태로 가능한 가장 큰 값을 찾습니다.

`mid`를 위쪽으로 치우치게 계산합니다.

```csharp
int mid = (left + right + 1) / 2;
```

그래야 가능한 값을 찾았을 때 `left = mid`로 옮겨도 무한 반복이 생기지 않습니다. ⚡

---

## 💻 C# 코드

```csharp
using System;

public class Solution
{
    public int solution(int distance, int[] rocks, int n)
    {
        Array.Sort(rocks);

        int left = 1, right = distance;

        while (left < right)
        {
            int mid = (left + right + 1) / 2;
            int prev = 0, removed = 0;

            foreach (int rock in rocks)
            {
                if (rock - prev < mid)
                    removed++;
                else
                    prev = rock;
            }

            if (distance - prev < mid)
                removed++;

            if (removed <= n)
                left = mid;
            else
                right = mid - 1;
        }

        return left;
    }
}
```

---

## ✨ 코드 의미

### 1. mid를 위쪽으로 계산

```csharp
int mid = (left + right + 1) / 2;
```

가능한 값을 찾으면 `left = mid`로 이동합니다.

이때 일반적인 `(left + right) / 2`를 쓰면 `left`와 `right`가 1 차이일 때 같은 값이 반복될 수 있습니다.

그래서 `+1`을 더해 위쪽 중간값을 사용합니다.

---

### 2. 가능한 거리면 left 이동

```csharp
if (removed <= n)
    left = mid;
```

`mid`가 가능하면 더 큰 값도 가능한지 봐야 합니다.

따라서 왼쪽 경계를 `mid`까지 올립니다.

---

### 3. 불가능하면 right 이동

```csharp
else
    right = mid - 1;
```

`mid`가 불가능하면 그보다 큰 값도 불가능합니다.

따라서 오른쪽 경계를 줄입니다.

---

## ⏱️ 시간 복잡도

풀이 1과 같습니다.

```text
O(r log r + r log D)
```

입니다.

---

## 📦 공간 복잡도

추가 배열이나 컬렉션을 만들지 않습니다.

```text
O(1)
```

입니다.

---

# ⚖️ 두 풀이 비교

| 풀이 | 핵심 방법 | 장점 | 시간 복잡도 | 공간 복잡도 |
|---|---|---|---:|---:|
| 풀이 1 | 정렬 + 이분 탐색 | 흐름이 명확하다 😊 | O(r log r + r log D) | O(1) |
| 풀이 2 | upper bound(가능한 가장 큰 값 찾기) 이분 탐색 | 코드가 더 짧다 🚀 | O(r log r + r log D) | O(1) |

# 🏆 추천 풀이

처음 공부할 때는 **풀이 1번**을 추천합니다.

이유는 다음과 같습니다.

```text
1. answer를 저장하는 기본 이분 탐색 흐름이 잘 보입니다.
2. 제거해야 하는 바위 수를 세는 과정이 명확합니다.
3. 가능한 값이면 더 크게 시도한다는 구조를 이해하기 좋습니다.
```

코딩 테스트 제출용으로 코드 길이를 우선한다면 **풀이 2번**도 좋습니다.

다만 `mid = (left + right + 1) / 2`를 쓰는 이유를 꼭 이해해야 합니다. 🌱

# ✅ 최종 정리

이 문제는 바위를 직접 조합해서 제거하는 문제가 아닙니다.

핵심은 다음입니다.

```text
최소 거리 x를 만들 수 있는지 판정한다.
```

판정 방법은 다음입니다.

```text
왼쪽부터 보면서, 직전에 남긴 지점과 거리가 x보다 짧은 바위는 제거한다.
```

이렇게 `x`가 가능한지 확인할 수 있으므로, 이분 탐색으로 가장 큰 `x`를 찾으면 됩니다.

🌉 “징검다리”는 정답을 직접 이분 탐색하는 대표적인 문제입니다.
