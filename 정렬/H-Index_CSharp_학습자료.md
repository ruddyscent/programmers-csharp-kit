# 📚 H-Index — C# 학습자료

## 📌 1. 문제 핵심

H-Index는 연구자의 논문 영향력을 나타내는 지표입니다.

문제에서 H-Index는 다음 조건을 만족하는 가장 큰 값 `h`입니다.

```text
h번 이상 인용된 논문이 h편 이상 있다.
```

예를 들어:

```text
citations = [3, 0, 6, 1, 5]
```

논문 인용 수를 정렬해 보면:

```text
[6, 5, 3, 1, 0]
```

이때:

```text
3회 이상 인용된 논문이 3편 있다.
```

따라서 H-Index는:

```text
3
```

입니다. ✅

---

## 🧠 핵심 아이디어

H-Index를 찾으려면 논문 인용 수를 큰 순서로 정렬하면 쉽습니다.

예를 들어:

```text
[6, 5, 3, 1, 0]
```

을 보면:

| 순서 | 인용 수 | 의미 |
|---:|---:|---|
| 1번째 | 6 | 1편 이상이 1회 이상 인용됨 |
| 2번째 | 5 | 2편 이상이 2회 이상 인용됨 |
| 3번째 | 3 | 3편 이상이 3회 이상 인용됨 |
| 4번째 | 1 | 4편 이상이 4회 이상 인용되지는 않음 |

3번째 논문의 인용 수가 3입니다.

즉:

```text
3편의 논문이 각각 3회 이상 인용됨
```

그래서 H-Index는 3입니다. 🎯

---

# 🔍 예제 이해하기

```text
citations = [3, 0, 6, 1, 5]
```

내림차순 정렬:

```text
[6, 5, 3, 1, 0]
```

하나씩 확인해 봅니다.

## h = 1

```text
1회 이상 인용된 논문이 1편 이상 있는가?
```

있습니다.

```text
6
```

가능합니다. ✅

## h = 2

```text
2회 이상 인용된 논문이 2편 이상 있는가?
```

있습니다.

```text
6, 5
```

가능합니다. ✅

## h = 3

```text
3회 이상 인용된 논문이 3편 이상 있는가?
```

있습니다.

```text
6, 5, 3
```

가능합니다. ✅

## h = 4

```text
4회 이상 인용된 논문이 4편 이상 있는가?
```

없습니다.

```text
6, 5
```

2편뿐입니다.

따라서 가장 큰 h는:

```text
3
```

입니다. 🎉

# ⚠️ 헷갈리기 쉬운 점

## 1. h는 인용 수가 아니라 조건을 만족하는 값

H-Index는 단순히 가장 많이 인용된 논문의 인용 수가 아닙니다.

예를 들어:

```text
[100, 1, 1, 1]
```

가 있다면 가장 많이 인용된 논문은 100회입니다.

하지만 H-Index는 1입니다.

왜냐하면:

```text
2회 이상 인용된 논문이 2편 이상 있지 않기 때문
```

입니다. 🚫

---

## 2. 나머지 논문 조건은 자연스럽게 만족됨

문제 설명에는 다음 표현이 있습니다.

```text
나머지 논문이 h번 이하 인용되었다면
```

하지만 보통 풀이에서는 다음 조건만 확인해도 충분합니다.

```text
h번 이상 인용된 논문이 h편 이상
```

정렬해서 가장 큰 h를 찾으면 나머지 조건은 자연스럽게 맞아떨어집니다. 😊

---

# 🧠 풀이 1. 이해하기 쉬운 방법 — 내림차순 정렬 후 확인하기

## 💡 아이디어

1. 인용 수 배열을 내림차순으로 정렬합니다.
2. 앞에서부터 확인합니다.
3. `i + 1`번째 논문의 인용 수가 `i + 1` 이상이면 H-Index 후보입니다.
4. 조건을 만족하는 가장 큰 값을 정답으로 저장합니다.

---

## 💻 C# 코드

```csharp
using System;

public class Solution
{
    public int solution(int[] citations)
    {
        Array.Sort(citations);
        Array.Reverse(citations);

        for (int i = 0; i < citations.Length; i++)
        {
            int h = i + 1;

            if (citations[i] < h)
                return i;
        }

        return citations.Length;
    }
}
```

---

## 🔎 코드 해설

### 1. 내림차순 정렬

```csharp
Array.Sort(citations);
Array.Reverse(citations);
```

인용 수가 큰 논문부터 확인하기 위해 내림차순으로 정렬합니다.

예:

```text
[3, 0, 6, 1, 5]
```

정렬 후:

```text
[6, 5, 3, 1, 0]
```

---

### 2. h 후보 만들기

```csharp
int h = i + 1;
```

배열 인덱스는 0부터 시작하지만, 논문 편수는 1부터 셉니다.

따라서:

```text
i = 0 → h = 1
i = 1 → h = 2
i = 2 → h = 3
```

입니다.

---

### 3. 조건 확인

```csharp
if (citations[i] < h)
```

내림차순으로 정렬했으므로 `i`번째 위치까지는 적어도 `h`편의 논문이 있습니다.

`citations[i] < h`라면:

```text
h번 이상 인용된 논문이 h편 이상 있다는 조건이 깨진다.
```

는 뜻입니다.

---

### 4. 조건이 깨지면 종료

```csharp
return i;
```

내림차순 정렬 상태에서 한 번 조건이 깨지면, 뒤쪽은 인용 수가 더 작습니다.

따라서 바로 직전 값인 `i`가 정답입니다.

---

## ⏱️ 시간 복잡도

정렬이 가장 큰 비용입니다.

```text
O(n log n)
```

`n`은 논문 수입니다.

---

## 📦 공간 복잡도

입력 배열을 직접 정렬합니다.

```text
O(1)
```

입니다.

# 🚀 풀이 2. 짧은 코드 버전 — 오름차순 정렬로 바로 반환하기

## 💡 아이디어

오름차순 정렬 후, 현재 위치에서 남은 논문 수를 H-Index 후보로 봅니다.

```text
남은 논문 수 = n - i
```

`citations[i] >= n - i`가 처음 성립하면 그 값이 정답입니다.

---

## 💻 C# 코드

```csharp
using System;

public class Solution
{
    public int solution(int[] citations)
    {
        Array.Sort(citations);

        int n = citations.Length;

        for (int i = 0; i < n; i++)
            if (citations[i] >= n - i)
                return n - i;

        return 0;
    }
}
```

---

## ✨ 코드 의미

### 1. 오름차순 정렬

```csharp
Array.Sort(citations);
```

인용 수가 작은 순서대로 정렬합니다.

---

### 2. 남은 논문 수를 후보로 보기

```csharp
n - i
```

오름차순 정렬에서 `i`번째부터 끝까지 남은 논문 수입니다.

---

### 3. 조건이 맞으면 바로 반환

```csharp
if (citations[i] >= n - i)
    return n - i;
```

`n - i`편 이상의 논문이 `n - i`번 이상 인용되었다는 뜻입니다. ✅

---

## ⚠️ 주의

이 풀이가 가능한 이유는 인용 수를 오름차순으로 정렬했기 때문입니다.

정렬하지 않고 `Count()`만 하면 안 됩니다. 🚫

---

## ⏱️ 시간 복잡도

정렬 때문에:

```text
O(n log n)
```

입니다.

---

## 📦 공간 복잡도

입력 배열을 직접 정렬합니다.

```text
O(1)
```

입니다.

# ✅ 최종 정리

H-Index는 다음 조건을 만족하는 가장 큰 값입니다.

```text
h번 이상 인용된 논문이 h편 이상 있다.
```

가장 쉬운 접근은:

```text
1. 인용 수를 내림차순 정렬한다.
2. i번째 논문이 i + 1회 이상 인용되었는지 확인한다.
3. 조건을 만족하는 최대 i + 1을 반환한다.
```

📚 “H-Index”는 정렬 후 위치와 인용 수를 비교하는 대표 문제입니다.
