# 🔢 가장 큰 수 — C# 학습자료

## 📌 1. 문제 핵심

0 또는 양의 정수들이 들어 있는 배열 `numbers`가 주어집니다.

이 숫자들을 어떤 순서로 이어 붙였을 때 만들 수 있는 **가장 큰 수**를 문자열로 반환해야 합니다.

예를 들어:

```text
numbers = [6, 10, 2]
```

가능한 조합 중 가장 큰 수는:

```text
6210
```

입니다. ✅

---

## ⚠️ 단순 숫자 정렬로는 안 되는 이유

숫자를 큰 순서로 정렬하면 될 것 같지만, 그렇지 않습니다.

예를 들어:

```text
[3, 30, 34, 5, 9]
```

숫자 크기만 보면:

```text
34 > 30 > 9 > 5 > 3
```

처럼 생각할 수 있습니다.

하지만 정답은:

```text
9534330
```

입니다.

왜냐하면 이 문제는 숫자의 크기가 아니라 **이어 붙였을 때 더 큰 문자열**을 만들어야 하기 때문입니다. 🎯

---

# 🧠 핵심 아이디어

두 숫자 `a`, `b`가 있을 때 어느 쪽을 앞에 둘지 비교합니다.

```text
a + b
b + a
```

둘 중 더 큰 쪽이 되도록 정렬합니다.

예를 들어 `3`과 `30`을 비교해 봅시다.

```text
"3" + "30" = "330"
"30" + "3" = "303"
```

`330`이 더 크므로 `3`이 `30`보다 앞에 와야 합니다.

```text
3, 30
```

✅

---

## 🔍 예제 1

```text
numbers = [6, 10, 2]
```

문자열로 바꾸면:

```text
["6", "10", "2"]
```

비교 기준은 다음입니다.

```text
"6" + "10" = "610"
"10" + "6" = "106"
```

`610`이 더 크므로 `6`이 `10`보다 앞입니다.

```text
"6" + "2" = "62"
"2" + "6" = "26"
```

`62`가 더 크므로 `6`이 `2`보다 앞입니다.

```text
"2" + "10" = "210"
"10" + "2" = "102"
```

`210`이 더 크므로 `2`가 `10`보다 앞입니다.

따라서 순서는:

```text
6, 2, 10
```

결과:

```text
"6210"
```

입니다. 🎉

---

## 🔍 예제 2

```text
numbers = [3, 30, 34, 5, 9]
```

올바른 정렬 순서는 다음입니다.

```text
9, 5, 34, 3, 30
```

이어 붙이면:

```text
"9534330"
```

입니다. ✅

---

# 🚨 예외 처리: 모든 숫자가 0인 경우

예를 들어:

```text
numbers = [0, 0, 0]
```

정렬 후 이어 붙이면:

```text
"000"
```

이 됩니다.

하지만 가장 큰 수는 `"000"`이 아니라:

```text
"0"
```

이어야 합니다.

따라서 정렬 후 첫 번째 값이 `"0"`이면 전체 결과는 `"0"`으로 반환합니다. ⚠️

---

# 🧠 풀이 1. 이해하기 쉬운 방법 — 문자열 배열 정렬하기

## 💡 아이디어

1. 숫자를 문자열로 변환합니다.
2. 두 문자열 `a`, `b`에 대해 `(b + a)`와 `(a + b)`를 비교합니다.
3. 더 큰 조합이 앞에 오도록 정렬합니다.
4. 정렬 결과를 이어 붙입니다.
5. 첫 값이 `"0"`이면 `"0"`을 반환합니다.

---

## 💻 C# 코드

```csharp
using System;

public class Solution
{
    public string solution(int[] numbers)
    {
        string[] nums = new string[numbers.Length];

        for (int i = 0; i < numbers.Length; i++)
            nums[i] = numbers[i].ToString();

        Array.Sort(nums, (a, b) => (b + a).CompareTo(a + b));

        if (nums[0] == "0")
            return "0";

        return string.Concat(nums);
    }
}
```

---

## 🔎 코드 해설

### 1. 숫자를 문자열로 변환

```csharp
nums[i] = numbers[i].ToString();
```

숫자를 이어 붙여 비교해야 하므로 문자열로 바꿉니다.

예를 들어:

```text
6 → "6"
10 → "10"
```

---

### 2. 커스텀 정렬

```csharp
Array.Sort(nums, (a, b) => (b + a).CompareTo(a + b));
```

이 부분이 핵심입니다.

`a`를 앞에 둘지, `b`를 앞에 둘지 비교합니다.

예를 들어 `a = "3"`, `b = "30"`이면:

```text
a + b = "330"
b + a = "303"
```

`a + b`가 더 크므로 `"3"`이 앞에 와야 합니다.

C#의 `Array.Sort` 비교 함수에서 음수가 나오면 `a`가 앞에 옵니다.  
그래서 내림차순 효과를 위해 `(b + a).CompareTo(a + b)`를 사용합니다. 🧠

---

### 3. 모든 값이 0인 경우 처리

```csharp
if (nums[0] == "0")
{
    return "0";
}
```

가장 앞 값이 `"0"`이라는 것은 모든 숫자가 0이라는 뜻입니다.

왜냐하면 가장 큰 조합이 앞에 오도록 정렬했는데도 첫 값이 0이면, 다른 값도 모두 0이기 때문입니다.

---

### 4. 문자열 이어 붙이기

```csharp
return string.Concat(nums);
```

정렬된 문자열 배열을 하나로 이어 붙입니다.

---

## ⏱️ 시간 복잡도

숫자의 개수를 `n`이라고 하면 정렬 비용이 가장 큽니다.

```text
O(n log n)
```

비교할 때 문자열을 이어 붙이므로 문자열 길이에 따른 비용도 있지만, 숫자는 최대 1000이라 길이가 짧습니다.

전체적으로:

```text
O(n log n)
```

으로 볼 수 있습니다.

---

## 📦 공간 복잡도

문자열 배열을 새로 만듭니다.

```text
O(n)
```

입니다.

# 🚀 풀이 2. 짧은 코드 버전 — Array.ConvertAll 사용하기

## 💡 아이디어

`Array.ConvertAll`로 숫자를 문자열로 바꾸고, 같은 비교 기준으로 정렬한 뒤 이어 붙입니다.

프로그래머스 제출용으로 안정적인 짧은 코드는 `Array.ConvertAll`과 `Array.Sort`를 함께 쓰는 방식입니다.

---

## 💻 C# 코드

```csharp
using System;

public class Solution
{
    public string solution(int[] numbers)
    {
        var nums = Array.ConvertAll(numbers, x => x.ToString());

        Array.Sort(nums, (a, b) => (b + a).CompareTo(a + b));

        return nums[0] == "0" ? "0" : string.Concat(nums);
    }
}
```

---

## ✨ 코드 의미

### 1. 한 줄로 문자열 배열 만들기

```csharp
var nums = Array.ConvertAll(numbers, x => x.ToString());
```

`numbers`의 모든 숫자를 문자열로 바꿉니다.

---

### 2. 가장 큰 수가 되도록 정렬

```csharp
Array.Sort(nums, (a, b) => (b + a).CompareTo(a + b));
```

두 문자열을 이어 붙였을 때 더 큰 조합이 앞에 오도록 정렬합니다.

---

### 3. 결과 반환

```csharp
return nums[0] == "0" ? "0" : string.Concat(nums);
```

모든 숫자가 0이면 `"0"`을 반환하고, 그렇지 않으면 이어 붙인 결과를 반환합니다. ✅

---

## ⏱️ 시간 복잡도

정렬 때문에:

```text
O(n log n)
```

입니다.

---

## 📦 공간 복잡도

문자열 배열을 사용합니다.

```text
O(n)
```

입니다.

# ✅ 최종 정리

이 문제의 핵심은 숫자 자체의 크기가 아닙니다.

```text
두 숫자를 어떤 순서로 이어 붙였을 때 더 큰가?
```

를 비교해야 합니다.

정렬 기준은 다음입니다.

```text
a + b 와 b + a 중 더 큰 쪽이 앞에 오게 한다.
```

그리고 모든 숫자가 0인 경우:

```text
"000"이 아니라 "0"
```

을 반환해야 합니다.

🔢 “가장 큰 수”는 문자열 정렬 기준을 직접 만드는 대표 문제입니다.
