# 🚤 구명보트 — C# 학습자료

## 📌 1. 문제 핵심

무인도에 갇힌 사람들을 구명보트로 구출해야 합니다.

구명보트에는 제한이 있습니다.

```text
1. 한 번에 최대 2명까지만 탈 수 있다.
2. 두 사람의 몸무게 합이 limit 이하여야 한다.
```

목표는 다음입니다.

```text
모든 사람을 구출하는 데 필요한 구명보트 개수의 최솟값 구하기
```

🎯

---

# 🔍 예제 1 이해하기

```text
people = [70, 50, 80, 50]
limit = 100
```

몸무게를 오름차순으로 정렬하면:

```text
[50, 50, 70, 80]
```

가장 무거운 사람 `80`은 가장 가벼운 사람 `50`과 함께 탈 수 없습니다.

```text
80 + 50 = 130 > 100
```

따라서 `80`은 혼자 탑니다.

```text
보트 1개: [80]
```

남은 사람:

```text
[50, 50, 70]
```

가장 무거운 사람 `70`도 가장 가벼운 사람 `50`과 함께 탈 수 없습니다.

```text
70 + 50 = 120 > 100
```

따라서 `70`도 혼자 탑니다.

```text
보트 1개: [70]
```

남은 사람:

```text
[50, 50]
```

두 사람은 함께 탈 수 있습니다.

```text
50 + 50 = 100
```

```text
보트 1개: [50, 50]
```

총 보트 수:

```text
3
```

✅

---

# 🧠 핵심 아이디어

가장 무거운 사람부터 태운다고 생각합니다.

이때 가장 무거운 사람이 가장 가벼운 사람과도 함께 탈 수 없다면, 다른 누구와도 함께 탈 수 없습니다.

왜냐하면 다른 사람들은 가장 가벼운 사람보다 무겁거나 같기 때문입니다.

즉:

```text
가장 무거운 사람 + 가장 가벼운 사람 > limit
```

이면 가장 무거운 사람은 혼자 타야 합니다. 🚤

반대로:

```text
가장 무거운 사람 + 가장 가벼운 사람 <= limit
```

이면 둘을 함께 태우는 것이 좋습니다.

가장 무거운 사람을 태우는 김에 가장 가벼운 사람도 같이 보내면 보트를 아낄 수 있습니다. 😊

---

# 🧩 필요한 개념: 투 포인터

정렬한 배열에서 양쪽 끝을 가리키는 두 포인터를 사용합니다.

```text
left  → 가장 가벼운 사람
right → 가장 무거운 사람
```

정렬된 배열:

```text
[50, 50, 70, 80]
 ↑           ↑
left       right
```

매번 가장 무거운 사람은 반드시 한 보트에 탑니다.

그리고 가장 가벼운 사람과 함께 탈 수 있으면 같이 태웁니다.

---

# 🧠 풀이 1. 이해하기 쉬운 방법 — 정렬 + 투 포인터

## 💡 아이디어

1. 사람들의 몸무게를 오름차순으로 정렬합니다.
2. `left`는 가장 가벼운 사람을 가리킵니다.
3. `right`는 가장 무거운 사람을 가리킵니다.
4. 두 사람이 함께 탈 수 있으면 둘 다 태웁니다.
5. 함께 탈 수 없으면 가장 무거운 사람만 태웁니다.
6. 보트 개수를 1 증가시킵니다.
7. 모든 사람이 탈 때까지 반복합니다.

---

## 💻 C# 코드

```csharp
using System;

public class Solution
{
    public int solution(int[] people, int limit)
    {
        Array.Sort(people);

        int left = 0;
        int right = people.Length - 1;
        int answer = 0;

        while (left <= right)
        {
            if (people[left] + people[right] <= limit)
            {
                left++;
            }

            right--;
            answer++;
        }

        return answer;
    }
}
```

---

## 🔎 코드 해설

### 1. 몸무게 정렬

```csharp
Array.Sort(people);
```

오름차순으로 정렬합니다.

예를 들어:

```text
[70, 50, 80, 50]
```

은:

```text
[50, 50, 70, 80]
```

이 됩니다.

---

### 2. 양쪽 포인터 설정

```csharp
int left = 0;
int right = people.Length - 1;
```

`left`는 가장 가벼운 사람입니다.

`right`는 가장 무거운 사람입니다.

---

### 3. 함께 탈 수 있는지 확인

```csharp
if (people[left] + people[right] <= limit)
{
    left++;
}
```

가장 가벼운 사람과 가장 무거운 사람이 함께 탈 수 있으면, 두 사람을 함께 태웁니다.

이때 가장 가벼운 사람도 구출되었으므로 `left++` 합니다.

---

### 4. 가장 무거운 사람은 항상 탄다

```csharp
right--;
answer++;
```

가장 무거운 사람은 혼자 타든, 가벼운 사람과 같이 타든 이번 보트에 반드시 탑니다.

그래서 항상 `right--` 합니다.

보트도 하나 사용했으므로 `answer++` 합니다. 🚤

---

### 5. 반복 조건

```csharp
while (left <= right)
```

아직 구출하지 않은 사람이 남아 있으면 계속 반복합니다.

`left == right`인 경우는 한 명만 남은 상황입니다.

이 사람도 보트 하나를 타야 합니다.

---

## ⏱️ 시간 복잡도

정렬이 가장 큰 비용입니다.

```text
O(n log n)
```

투 포인터 순회는 한 번만 합니다.

```text
O(n)
```

전체 시간 복잡도는:

```text
O(n log n)
```

입니다.

---

## 📦 공간 복잡도

추가 배열을 만들지 않습니다.

```text
O(1)
```

단, 정렬 내부 구현에 따라 약간의 추가 공간이 사용될 수 있습니다.

# 🚀 풀이 2. 코드가 짧은 방법 — LINQ로 정렬 후 투 포인터

## 💡 아이디어

풀이 1과 같은 방식입니다.

다만 원본 배열을 바꾸지 않고 LINQ의 `OrderBy`로 정렬된 배열을 만들어 사용할 수 있습니다.

---

## 💻 C# 코드

```csharp
using System;
using System.Linq;

public class Solution
{
    public int solution(int[] people, int limit)
    {
        int[] sorted = people.OrderBy(x => x).ToArray();

        int left = 0;
        int right = sorted.Length - 1;
        int boats = 0;

        while (left <= right)
        {
            if (sorted[left] + sorted[right] <= limit)
            {
                left++;
            }

            right--;
            boats++;
        }

        return boats;
    }
}
```

---

## ✨ 코드 의미

### 1. LINQ로 정렬

```csharp
int[] sorted = people.OrderBy(x => x).ToArray();
```

몸무게를 오름차순으로 정렬한 새 배열을 만듭니다.

원본 배열 `people`은 변경하지 않습니다.

---

### 2. 투 포인터 사용

```csharp
int left = 0;
int right = sorted.Length - 1;
```

가장 가벼운 사람과 가장 무거운 사람을 가리킵니다.

---

### 3. 가장 무거운 사람 기준으로 보트 사용

```csharp
if (sorted[left] + sorted[right] <= limit)
{
    left++;
}

right--;
boats++;
```

가장 무거운 사람은 반드시 이번 보트에 탑니다.

가장 가벼운 사람과 함께 탈 수 있으면 같이 태웁니다. ✅

---

## ⚠️ 프로그래머스에서 실행할 때 주의

LINQ를 사용하므로 다음 네임스페이스가 필요합니다.

```csharp
using System.Linq;
```

다만 입력 크기가 최대 50,000이므로 `Array.Sort`를 쓰는 풀이 1이 메모리 측면에서는 조금 더 좋습니다.

---

## ⏱️ 시간 복잡도

LINQ 정렬 비용은:

```text
O(n log n)
```

투 포인터 순회는:

```text
O(n)
```

전체 시간 복잡도는:

```text
O(n log n)
```

입니다.

---

## 📦 공간 복잡도

정렬된 새 배열을 만듭니다.

```text
O(n)
```

입니다.

# ❌ 왜 가장 가벼운 사람끼리 먼저 태우면 안 될까?

예를 들어:

```text
people = [40, 50, 60, 90]
limit = 100
```

가벼운 사람끼리 먼저 태우면:

```text
40 + 50 = 90
```

남은 사람:

```text
60, 90
```

이 둘은 같이 탈 수 없습니다.

총 보트:

```text
[40, 50], [60], [90] → 3개
```

하지만 가장 무거운 사람부터 보면:

```text
90 + 40 = 130 > 100 → 90 혼자
60 + 40 = 100 → 같이
50 혼자
```

여기서도 3개지만, 더 일반적으로 가장 무거운 사람을 기준으로 판단하는 것이 안전합니다.

왜냐하면 가장 무거운 사람이 가장 가벼운 사람과도 못 타면 무조건 혼자 타야 한다는 결론이 확실하기 때문입니다. 🎯

프로그래머스 제출용으로는 **풀이 1번 `Array.Sort` 방식**을 추천합니다.

이유는 다음과 같습니다.

```text
1. 입력 크기가 최대 50,000이라 정렬 효율이 중요합니다.
2. 새 배열을 만들지 않아 메모리를 덜 씁니다.

```

LINQ에 익숙하고 원본 배열을 보존하고 싶다면 **풀이 2번**도 좋습니다.

# ✅ 최종 정리

이 문제의 핵심은 다음입니다.

```text
가장 무거운 사람을 기준으로 보트를 배정한다.
```

정렬 후:

```text
가장 가벼운 사람 + 가장 무거운 사람 <= limit
```

이면 둘을 함께 태우고,

```text
초과하면 가장 무거운 사람만 태운다.
```

풀이 흐름은 다음과 같습니다.

```text
1. 몸무게를 오름차순 정렬한다.
2. left는 가장 가벼운 사람, right는 가장 무거운 사람을 가리킨다.
3. 둘이 함께 탈 수 있으면 left를 증가시킨다.
4. 가장 무거운 사람은 항상 태우므로 right를 감소시킨다.
5. 보트 개수를 증가시킨다.
```

🚤 “구명보트”는 정렬과 투 포인터를 함께 사용하는 대표적인 그리디 문제입니다.
