# 📸 단속카메라 — C# 학습자료

## 📌 1. 문제 핵심

각 차량은 고속도로의 특정 구간을 지나갑니다.

예를 들어:

```text
[-20, -15]
```

는 차량이 `-20` 지점에서 진입해서 `-15` 지점에서 나갔다는 뜻입니다.

카메라는 차량의 이동 구간 안에 설치되어 있으면 그 차량을 단속할 수 있습니다.

즉, 차량 경로가:

```text
[start, end]
```

라면 카메라 위치 `camera`가 다음 조건을 만족하면 됩니다.

```text
start <= camera <= end
```

목표는 다음입니다.

```text
모든 차량을 최소한 한 번씩 만나도록 하는 카메라 최소 개수 구하기
```

🎯

---

# 🔍 예제 이해하기

```text
routes = [[-20,-15], [-14,-5], [-18,-13], [-5,-3]]
```

차량 경로를 보면:

```text
1번 차량: -20 ~ -15
2번 차량: -14 ~ -5
3번 차량: -18 ~ -13
4번 차량: -5 ~ -3
```

카메라를 `-15`에 설치하면:

```text
1번 차량: -20 ~ -15 → 만남
3번 차량: -18 ~ -13 → 만남
```

카메라를 `-5`에 설치하면:

```text
2번 차량: -14 ~ -5 → 만남
4번 차량: -5 ~ -3 → 만남
```

따라서 카메라 2대면 모든 차량을 단속할 수 있습니다.

```text
return 2
```

✅

---

# 🧠 핵심 아이디어

이 문제는 구간을 최소 점 개수로 덮는 문제입니다.

가장 좋은 전략은:

```text
진출 지점이 빠른 차량부터 확인한다.
아직 카메라를 만나지 못한 차량이면,
그 차량의 진출 지점에 카메라를 설치한다.
```

입니다.

왜 진출 지점에 설치할까요?

현재 차량을 단속하려면 카메라는 반드시 현재 차량 구간 안에 있어야 합니다.

그중에서도 가능한 한 오른쪽, 즉 진출 지점에 설치하면 이후 차량들도 함께 커버할 가능성이 커집니다. 🚗

---

# 🧩 그리디 전략

1. 차량 경로를 진출 지점 기준으로 오름차순 정렬합니다.
2. 카메라 위치를 아주 작은 값으로 시작합니다.
3. 차량을 하나씩 확인합니다.
4. 현재 카메라가 차량 구간 안에 있으면 새 카메라는 필요 없습니다.
5. 현재 카메라가 차량 구간 밖에 있으면, 해당 차량의 진출 지점에 새 카메라를 설치합니다.

---

# 🔍 예제 흐름

입력:

```text
[[-20,-15], [-14,-5], [-18,-13], [-5,-3]]
```

진출 지점 기준으로 정렬하면:

```text
[-20, -15]
[-18, -13]
[-14, -5]
[-5, -3]
```

## 1번 차량: [-20, -15]

아직 카메라가 없습니다.

따라서 진출 지점 `-15`에 카메라 설치

```text
camera = -15
answer = 1
```

## 2번 차량: [-18, -13]

현재 카메라 `-15`는 이 차량 구간 안에 있습니다.

```text
-18 <= -15 <= -13
```

새 카메라 필요 없음 ✅

## 3번 차량: [-14, -5]

현재 카메라 `-15`는 이 차량 구간보다 왼쪽에 있습니다.

```text
-15 < -14
```

새 카메라가 필요합니다.

진출 지점 `-5`에 설치합니다.

```text
camera = -5
answer = 2
```

## 4번 차량: [-5, -3]

현재 카메라 `-5`는 이 차량 구간 안에 있습니다.

```text
-5 <= -5 <= -3
```

새 카메라 필요 없음 ✅

최종 정답:

```text
2
```

🎉

# 🧠 풀이 1. 이해하기 쉬운 방법 — 진출 지점 기준 정렬

## 💡 아이디어

1. 차량 경로를 진출 지점 기준으로 정렬합니다.
2. 현재 카메라 위치를 매우 작은 값으로 둡니다.
3. 각 차량을 확인합니다.
4. 현재 카메라가 차량의 진입 지점보다 작다면, 이 차량은 아직 카메라를 만나지 못한 것입니다.
5. 이 차량의 진출 지점에 카메라를 설치합니다.

---

## 💻 C# 코드

```csharp
using System;

public class Solution
{
    public int solution(int[,] routes)
    {
        int carCount = routes.GetLength(0);

        int[][] sortedRoutes = new int[carCount][];

        for (int i = 0; i < carCount; i++)
        {
            sortedRoutes[i] = new int[]
            {
                routes[i, 0],
                routes[i, 1]
            };
        }

        Array.Sort(sortedRoutes, (a, b) => a[1].CompareTo(b[1]));

        int camera = int.MinValue;
        int answer = 0;

        foreach (int[] route in sortedRoutes)
        {
            int start = route[0];
            int end = route[1];

            if (camera < start)
            {
                camera = end;
                answer++;
            }
        }

        return answer;
    }
}
```

---

## 🔎 코드 해설

### 1. 2차원 배열을 정렬하기 쉬운 형태로 변환

```csharp
int[][] sortedRoutes = new int[carCount][];
```

프로그래머스 C#에서 `routes`는 보통 `int[,]` 형태입니다.

`Array.Sort`로 행 단위 정렬을 쉽게 하기 위해 `int[][]`로 바꿉니다.

---

### 2. 진입 지점과 진출 지점 저장

```csharp
sortedRoutes[i] = new int[]
{
    routes[i, 0],
    routes[i, 1]
};
```

각 차량의 경로를 `[start, end]` 형태로 저장합니다.

---

### 3. 진출 지점 기준 정렬

```csharp
Array.Sort(sortedRoutes, (a, b) => a[1].CompareTo(b[1]));
```

진출 지점이 빠른 차량부터 처리합니다.

이것이 그리디의 핵심입니다. 🎯

---

### 4. 현재 카메라 위치

```csharp
int camera = int.MinValue;
```

아직 카메라가 없다고 생각하기 위해 아주 작은 값으로 시작합니다.

---

### 5. 카메라가 현재 차량을 커버하는지 확인

```csharp
if (camera < start)
```

현재 카메라가 차량의 진입 지점보다 왼쪽에 있다면, 이 차량은 카메라를 만나지 못합니다.

따라서 새 카메라가 필요합니다.

---

### 6. 진출 지점에 카메라 설치

```csharp
camera = end;
answer++;
```

현재 차량을 커버하면서, 이후 차량도 최대한 많이 커버하기 위해 진출 지점에 카메라를 설치합니다.

---

## ⏱️ 시간 복잡도

차량 수를 `n`이라고 하면 정렬이 가장 큰 비용입니다.

```text
O(n log n)
```

정렬 후 한 번 순회합니다.

```text
O(n)
```

전체 시간 복잡도는:

```text
O(n log n)
```

입니다.

---

## 📦 공간 복잡도

정렬을 위해 `int[][]` 배열을 새로 만듭니다.

```text
O(n)
```

입니다.

# 🚀 풀이 2. 코드가 짧은 방법 — LINQ로 정렬하기

## 💡 아이디어

LINQ를 사용해서 `routes`의 행 번호를 만들고, 각 차량 경로를 튜플로 변환한 뒤 진출 지점 기준으로 정렬합니다.

---

## 💻 C# 코드

```csharp
using System;
using System.Linq;

public class Solution
{
    public int solution(int[,] routes)
    {
        var sortedRoutes = Enumerable.Range(0, routes.GetLength(0))
            .Select(i => (Start: routes[i, 0], End: routes[i, 1]))
            .OrderBy(route => route.End);

        int camera = int.MinValue;
        int answer = 0;

        foreach (var route in sortedRoutes)
        {
            if (camera < route.Start)
            {
                camera = route.End;
                answer++;
            }
        }

        return answer;
    }
}
```

---

## ✨ 코드 의미

### 1. 행 번호 만들기

```csharp
Enumerable.Range(0, routes.GetLength(0))
```

차량 수만큼 행 번호를 만듭니다.

예를 들어 차량이 4대라면:

```text
0, 1, 2, 3
```

입니다.

---

### 2. 튜플로 변환

```csharp
.Select(i => (Start: routes[i, 0], End: routes[i, 1]))
```

각 차량 경로를 이름 있는 튜플로 바꿉니다.

```text
(Start, End)
```

---

### 3. 진출 지점 기준 정렬

```csharp
.OrderBy(route => route.End)
```

진출 지점이 빠른 순서대로 정렬합니다.

---

### 4. 카메라 설치 판단

```csharp
if (camera < route.Start)
```

현재 카메라가 이 차량의 진입 지점보다 왼쪽에 있으면 새 카메라가 필요합니다.

---

## ⚠️ 프로그래머스에서 실행할 때 주의

LINQ를 사용하므로 다음 네임스페이스가 필요합니다.

```csharp
using System.Linq;
```

프로그래머스 C# 환경에서 실행 가능합니다. ✅

---

## ⏱️ 시간 복잡도

정렬 때문에:

```text
O(n log n)
```

입니다.

---

## 📦 공간 복잡도

LINQ 정렬 결과를 저장하는 과정에서 추가 공간이 사용됩니다.

```text
O(n)
```

입니다.

# ❌ 잘못된 접근: 진입 지점 기준 정렬

진입 지점 기준으로 정렬하면 직관적으로 보일 수 있지만, 최소 카메라 개수를 보장하기 어렵습니다.

이 문제의 핵심은:

```text
가장 빨리 끝나는 차량을 먼저 확실히 커버한다.
```

입니다.

그래서 진출 지점 기준 정렬이 필요합니다. 🚦

# ⚖️ 두 풀이 비교

| 풀이 | 핵심 방법 | 장점 | 시간 복잡도 | 공간 복잡도 |
|---|---|---|---:|---:|
| 풀이 1 | 배열 변환 + 진출 지점 정렬 | 흐름이 명확하다 😊 | O(n log n) | O(n) |
| 풀이 2 | LINQ + 튜플 정렬 | 코드가 짧고 읽기 쉽다 🚀 | O(n log n) | O(n) |

# ✅ 최종 정리

이 문제의 핵심은 다음입니다.

```text
진출 지점이 빠른 차량부터 처리한다.
```

그리고 아직 카메라를 만나지 못한 차량이 나오면:

```text

```

풀이 흐름은 다음과 같습니다.

```text
1. 차량 경로를 진출 지점 기준으로 정렬한다.
2. 현재 카메라 위치를 저장한다.
3. 현재 차량의 진입 지점보다 카메라가 왼쪽에 있으면 새 카메라가 필요하다.
4. 새 카메라는 현재 차량의 진출 지점에 설치한다.
5. 카메라 수를 증가시킨다.
```

📸 “단속카메라”는 구간 정렬과 그리디를 함께 연습하기 좋은 대표 문제입니다.
