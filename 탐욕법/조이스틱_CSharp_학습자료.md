# 🕹️ 조이스틱 — C# 학습자료

## 📌 1. 문제 핵심

처음 이름은 모두 `A`로 이루어져 있습니다.

예를 들어 목표 이름이 `"JAN"`이면 처음 상태는:

```text
AAA
```

입니다.

조이스틱으로 해야 할 일은 두 가지입니다.

```text
1. 위/아래로 움직여 알파벳을 바꾼다.
2. 왼쪽/오른쪽으로 움직여 커서를 이동한다.
```

최종 목표는 원하는 이름을 만들 때 필요한 조작 횟수의 최솟값을 구하는 것입니다. 🎯

---

# 🧠 조작은 두 종류로 나뉜다

## 1. 알파벳 변경: 상하 이동

현재 문자는 항상 `A`에서 시작합니다.

예를 들어 `J`를 만들려면:

```text
A → B → C → ... → J
```

위로 9번 이동하면 됩니다.

하지만 `Z`는 위로 25번 가는 것보다 아래로 1번 가는 게 빠릅니다.

```text
A → Z
```

아래로 1번입니다. 🔽

따라서 어떤 문자 `ch`를 만들기 위한 최소 조작 횟수는:

```text
Math.Min(ch - 'A', 'Z' - ch + 1)
```

입니다.

---

## 2. 커서 이동: 좌우 이동

커서는 처음에 0번 위치에 있습니다.

가장 단순한 방법은 오른쪽으로 끝까지 이동하는 것입니다.

이 경우 이동 횟수는:

```text
name.Length - 1
```

입니다.

예를 들어 이름 길이가 6이면:

```text
0 → 1 → 2 → 3 → 4 → 5
```

총 5번 이동합니다.

하지만 중간에 `A`가 길게 이어져 있다면 굳이 그 구간을 지나갈 필요가 없을 수 있습니다. 🧭

---

# 🔍 예제: JAZ

```text
name = "JAZ"
```

처음 상태:

```text
AAA
```

## 1. 첫 번째 문자 J 만들기

```text
A → J
```

위로 9번입니다.

```text
조작 횟수: 9
```

## 2. 마지막 문자 Z로 이동

현재 커서는 첫 번째 위치입니다.

오른쪽으로 가면:

```text
0 → 1 → 2
```

2번 이동해야 합니다.

하지만 왼쪽으로 가면 마지막 위치로 바로 이동합니다.

```text
0 → 2
```

1번 이동입니다. 👈

## 3. Z 만들기

```text
A → Z
```

아래로 1번입니다.

총 조작 횟수:

```text
9 + 1 + 1 = 11
```

✅

# ⚠️ 어려운 부분: 연속된 A 피하기

예를 들어:

```text
BAAAAB
```

처럼 중간에 `A`가 많이 있으면, 오른쪽으로 끝까지 가는 것보다 되돌아가거나 왼쪽으로 도는 편이 더 짧을 수 있습니다.

커서 이동은 다음 세 가지를 고려해야 합니다.

```text
1. 그냥 오른쪽으로 쭉 가기
2. 오른쪽으로 갔다가 왼쪽으로 되돌아가기
3. 왼쪽으로 갔다가 오른쪽으로 되돌아가기
```

이 중 가장 작은 값을 선택해야 합니다. 🎯

# 🧠 풀이 1. 이해하기 쉬운 방법 — 상하 이동 + 좌우 최적화

## 💡 아이디어

전체 답은 다음 두 부분의 합입니다.

```text
답 = 알파벳 변경 횟수 + 커서 이동 횟수
```

알파벳 변경 횟수는 모든 문자에 대해 바로 계산할 수 있습니다.

커서 이동 횟수는 기본값을 오른쪽으로 끝까지 가는 것으로 둡니다.

```text
move = name.Length - 1
```

그다음 각 위치에서 뒤쪽에 연속된 `A` 구간이 있는지 확인하고, 그 구간을 피하는 이동 방법을 계산합니다.

---

## 💻 C# 코드

```csharp
using System;

public class Solution
{
    public int solution(string name)
    {
        int n = name.Length, answer = 0, move = n - 1;

        for (int i = 0; i < n; i++)
        {
            char ch = name[i];

            answer += Math.Min(ch - 'A', 'Z' - ch + 1);

            int next = i + 1;

            while (next < n && name[next] == 'A')
                next++;

            int goRightThenBack = i * 2 + n - next;
            int goLeftThenBack = i + (n - next) * 2;

            move = Math.Min(move, Math.Min(goRightThenBack, goLeftThenBack));
        }

        return answer + move;
    }
}
```

---

## 🔎 코드 해설

### 1. 기본 커서 이동

```csharp
int move = n - 1;
```

가장 단순한 방법은 오른쪽으로 끝까지 이동하는 것입니다.

이동 횟수는 `n - 1`입니다.

---

### 2. 알파벳 변경 횟수 계산

```csharp
answer += Math.Min(ch - 'A', 'Z' - ch + 1);
```

예를 들어 `J`라면:

```text
J - A = 9
Z - J + 1 = 17
```

따라서 9번이 최소입니다.

`Z`라면:

```text
Z - A = 25
Z - Z + 1 = 1
```

따라서 1번이 최소입니다. 🔽

---

### 3. 다음으로 A가 아닌 위치 찾기

```csharp
int next = i + 1;

while (next < n && name[next] == 'A')
{
    next++;
}
```

현재 위치 `i` 다음부터 연속된 `A`를 건너뜁니다.

예를 들어:

```text
B A A A A B
  ↑ ↑ ↑ ↑
```

이런 `A` 구간은 굳이 방문하지 않아도 됩니다.

---

### 4. 오른쪽으로 갔다가 되돌아가기

```csharp
int goRightThenBack = i * 2 + n - next;
```

의미는 다음과 같습니다.

```text
처음부터 i까지 오른쪽으로 간다.
다시 왼쪽으로 되돌아간다.
끝쪽의 A가 아닌 문자로 간다.
```

즉, 오른쪽으로 갔다가 방향을 바꾸는 경우입니다. ↪️

---

### 5. 왼쪽으로 갔다가 되돌아가기

```csharp
int goLeftThenBack = i + (n - next) * 2;
```

의미는 다음과 같습니다.

```text
처음부터 왼쪽 방향으로 끝쪽을 먼저 처리한다.
다시 오른쪽으로 돌아와 앞쪽을 처리한다.
```

반대 방향으로 도는 경우입니다. ↩️

---

### 6. 최소 이동 갱신

```csharp
move = Math.Min(move, Math.Min(goRightThenBack, goLeftThenBack));
```

기본 오른쪽 이동, 오른쪽 후 되돌아가기, 왼쪽 후 되돌아가기 중 가장 작은 값을 선택합니다.

---

## ⏱️ 시간 복잡도

겉으로는 반복문 안에 `while`이 있어 복잡해 보입니다.

하지만 이름 길이는 최대 20입니다.

실제로는 매우 작아서 충분히 빠릅니다.

일반적으로는:

```text
O(n²)
```

까지 볼 수 있지만, 제한상 문제없습니다.

조금 더 넓게 보면 `n <= 20`이라 사실상 상수 시간처럼 동작합니다. ✅

---

## 📦 공간 복잡도

추가 배열을 사용하지 않습니다.

```text
O(1)
```

입니다.

# 🚀 풀이 2. 짧은 코드 버전 — 같은 로직을 간결하게 작성하기

## 💡 아이디어

풀이 1과 같은 방식입니다.

다만 변수명을 줄이고 코드를 조금 더 압축합니다.

코딩 테스트 제출용으로 바로 사용할 수 있습니다. ✅

---

## 💻 C# 코드

```csharp
using System;

public class Solution
{
    public int solution(string name)
    {
        int n = name.Length, answer = 0, move = n - 1;

        for (int i = 0; i < n; i++)
        {
            answer += Math.Min(name[i] - 'A', 'Z' - name[i] + 1);

            int next = i + 1;

            while (next < n && name[next] == 'A')
                next++;

            move = Math.Min(move, Math.Min(i * 2 + n - next, i + 2 * (n - next)));
        }

        return answer + move;
    }
}
```

---

## ✨ 코드 의미

### 1. 상하 이동 누적

```csharp
answer += Math.Min(name[i] - 'A', 'Z' - name[i] + 1);
```

각 문자를 `A`에서 목표 문자로 바꾸는 최소 횟수를 더합니다.

---

### 2. 연속된 A 구간 찾기

```csharp
while (next < n && name[next] == 'A')
{
    next++;
}
```

현재 위치 뒤쪽에 있는 연속된 `A`를 건너뜁니다.

---

### 3. 좌우 이동 최솟값 갱신

```csharp
move = Math.Min(move, i * 2 + n - next);
move = Math.Min(move, i + 2 * (n - next));
```

두 방향 전환 경우를 각각 비교합니다.

```text
오른쪽으로 갔다가 되돌아가기
왼쪽으로 갔다가 되돌아가기
```

입니다. 🧭

---

## ⚠️ LINQ를 적극적으로 쓰지 않은 이유

알파벳 변경 횟수만 보면 LINQ로 쓸 수 있습니다.

```csharp
name.Sum(ch => Math.Min(ch - 'A', 'Z' - ch + 1))
```

하지만 커서 이동 최적화는 인덱스와 연속된 `A` 구간을 함께 봐야 합니다.

그래서 LINQ를 억지로 쓰면 오히려 이해하기 어려워집니다.

이 문제는 반복문이 가장 안전합니다. 🌱

---

## ⏱️ 시간 복잡도

```text
O(n²)
```

로 볼 수 있습니다.

하지만 `n <= 20`이므로 매우 빠릅니다. ⚡

---

## 📦 공간 복잡도

```text
O(1)
```

입니다.

## 상하 이동

```text
J: A → J = 9
A: 0
N: A → N = 13
```

상하 이동 합:

```text
9 + 0 + 13 = 22
```

## 좌우 이동

처음 위치는 0번입니다.

`J`를 만들고 오른쪽으로 1번 이동하면 `A`입니다.

그다음 오른쪽으로 1번 더 가면 `N`입니다.

하지만 더 좋은 방법은:

```text
0번에서 왼쪽으로 1번 이동 → 마지막 위치 N
```

입니다.

좌우 이동 최소:

```text
1
```

총 조작 횟수:

```text
22 + 1 = 23
```

정답:

```text
23
```

✅

# ✅ 최종 정리

이 문제는 두 가지 비용을 더해야 합니다.

```text
1. 각 문자를 A에서 목표 문자로 바꾸는 비용
2. 커서를 좌우로 움직이는 비용
```

상하 이동은 간단합니다.

```text
Math.Min(ch - 'A', 'Z' - ch + 1)
```

좌우 이동은 연속된 `A` 구간을 피해 최소 이동을 찾아야 합니다.

```text
오른쪽으로 쭉 가기
오른쪽으로 갔다가 되돌아가기
왼쪽으로 갔다가 되돌아가기
```

세 경우 중 가장 작은 값을 고릅니다.

🕹️ “조이스틱”은 그리디 문제 중에서도 좌우 이동 최적화가 핵심인 대표 문제입니다.
