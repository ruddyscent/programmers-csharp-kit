# 🔢 큰 수 만들기 — C# 학습자료

## 📌 1. 문제 핵심

숫자로 이루어진 문자열 `number`가 주어집니다.

여기서 숫자 `k`개를 제거해서 만들 수 있는 가장 큰 수를 구해야 합니다.

예를 들어:

```text
number = "1924"
k = 2
```

숫자 2개를 제거해서 만들 수 있는 수는 다음과 같습니다.

```text
19, 12, 14, 92, 94, 24
```

이 중 가장 큰 수는:

```text
94
```

입니다. ✅

---

# ⚠️ 조합으로 풀면 안 되는 이유

가장 먼저 떠올릴 수 있는 방법은 숫자 `k`개를 제거하는 모든 경우를 만들어 보는 것입니다.

하지만 `number`의 길이는 최대:

```text
1,000,000
```

입니다.

모든 조합을 만들면 경우의 수가 너무 커져서 시간 초과가 납니다. 🚨

따라서 이 문제는 조합 문제가 아니라 **그리디 문제**로 풀어야 합니다.

---

# 🧠 핵심 아이디어

왼쪽부터 숫자를 보면서, 앞에 있는 숫자가 현재 숫자보다 작으면 제거하는 것이 유리합니다.

예를 들어:

```text
number = "1924"
```

처음 `1` 다음에 `9`가 나옵니다.

```text
1 9
```

큰 수를 만들려면 앞자리가 클수록 좋습니다.

따라서 `1`을 제거하고 `9`를 앞에 두는 것이 좋습니다.

```text
1924 → 924
```

이런 식으로:

```text
앞 숫자가 뒤 숫자보다 작으면 제거한다.
```

가 핵심입니다. 🎯

---

# 🧺 스택으로 생각하기

결과 숫자를 담을 공간을 스택처럼 사용합니다.

숫자를 하나씩 보면서:

```text
1. 현재 숫자보다 작은 숫자가 결과 끝에 있으면 제거한다.
2. 제거할 수 있는 횟수 k를 1 줄인다.
3. 현재 숫자를 결과에 넣는다.
```

이렇게 하면 큰 숫자가 앞쪽에 오도록 만들 수 있습니다.

---

# 🔍 예제 1: "1924", k = 2

```text
number = "1924"
k = 2
```

## 1 처리

```text
결과: 1
```

## 9 처리

현재 숫자 `9`가 결과 끝의 `1`보다 큽니다.

`1`을 제거합니다.

```text
결과: 
k = 1
```

`9`를 넣습니다.

```text
결과: 9
```

## 2 처리

`2`는 `9`보다 작으므로 그냥 넣습니다.

```text
결과: 92
```

## 4 처리

현재 숫자 `4`가 결과 끝의 `2`보다 큽니다.

`2`를 제거합니다.

```text
결과: 9
k = 0
```

`4`를 넣습니다.

```text
결과: 94
```

정답:

```text
94
```

🎉

# 🧠 풀이 1. 이해하기 쉬운 방법 — List를 스택처럼 사용하기

## 💡 아이디어

1. 결과를 담을 `List<char>`를 만듭니다.
2. 숫자를 왼쪽부터 하나씩 봅니다.
3. 결과의 마지막 숫자가 현재 숫자보다 작고, 아직 제거할 수 있으면 제거합니다.
4. 현재 숫자를 결과에 추가합니다.
5. 숫자를 다 봤는데 제거 횟수가 남으면 뒤에서 제거합니다.
6. 필요한 길이만큼 문자열로 만들어 반환합니다.

---

## 💻 C# 코드

```csharp
using System.Collections.Generic;

public class Solution
{
    public string solution(string number, int k)
    {
        int removeCount = k;
        List<char> result = new List<char>();

        foreach (char digit in number)
        {
            while (removeCount > 0 && result.Count > 0 && result[result.Count - 1] < digit)
            {
                result.RemoveAt(result.Count - 1);
                removeCount--;
            }

            result.Add(digit);
        }

        result.RemoveRange(result.Count - removeCount, removeCount);

        return new string(result.ToArray());
    }
}
```

---

## 🔎 코드 해설

### 1. 제거 횟수 저장

```csharp
int removeCount = k;
```

앞으로 제거할 수 있는 숫자의 개수입니다.

---

### 2. 결과를 담을 List

```csharp
List<char> result = new List<char>();
```

숫자를 하나씩 넣고, 필요하면 마지막 숫자를 제거합니다.

스택처럼 사용합니다. 🧺

---

### 3. 앞 숫자가 현재 숫자보다 작으면 제거

```csharp
while (removeCount > 0 &&
       result.Count > 0 &&
       result[result.Count - 1] < digit)
```

조건은 세 가지입니다.

```text
1. 아직 제거할 수 있어야 한다.
2. 결과에 숫자가 있어야 한다.
3. 결과 마지막 숫자가 현재 숫자보다 작아야 한다.
```

이 조건을 만족하면 결과의 마지막 숫자를 제거합니다.

---

### 4. 마지막 숫자 제거

```csharp
result.RemoveAt(result.Count - 1);
removeCount--;
```

작은 앞 숫자를 제거하면 더 큰 숫자가 앞쪽으로 갈 수 있습니다.

---

### 5. 현재 숫자 추가

```csharp
result.Add(digit);
```

현재 숫자를 결과에 넣습니다.

---

### 6. 제거 횟수가 남은 경우

```csharp
result.RemoveRange(result.Count - removeCount, removeCount);
```

예를 들어 숫자가 내림차순이면 앞 숫자를 제거할 기회가 없습니다.

```text
987654
```

이 경우 가장 큰 수를 만들려면 뒤에서 제거해야 합니다.

---

## ⏱️ 시간 복잡도

각 숫자는 한 번 추가되고, 많아야 한 번 제거됩니다.

따라서 전체 시간 복잡도는:

```text
O(n)
```

입니다.

`n`은 `number.Length`입니다.

---

## 📦 공간 복잡도

결과를 저장하는 List를 사용합니다.

```text
O(n)
```

입니다.

# 🚀 풀이 2. 짧은 코드 버전 — char 배열을 스택처럼 사용하기

## 💡 아이디어

`List<char>`도 충분히 통과할 수 있지만, `number` 길이가 최대 1,000,000이므로 더 빠르게 하려면 `char[]`를 직접 스택처럼 사용할 수 있습니다.

`top`은 현재 스택의 길이를 의미합니다.

```text
stack[0..top-1]이 현재 결과
```

---

## 💻 C# 코드

```csharp
public class Solution
{
    public string solution(string number, int k)
    {
        int keepLength = number.Length - k, top = 0;
        char[] stack = new char[number.Length];

        foreach (char digit in number)
        {
            while (k > 0 && top > 0 && stack[top - 1] < digit)
            {
                top--;
                k--;
            }

            stack[top++] = digit;
        }

        return new string(stack, 0, keepLength);
    }
}
```

---

## ✨ 코드 의미

### 1. 최종 길이 계산

```csharp
int keepLength = number.Length - k;
```

`k`개를 제거하므로 최종 문자열 길이는:

```text
number.Length - k
```

입니다.

예를 들어:

```text
"1924", k = 2
```

이면 최종 길이는:

```text
2
```

입니다.

---

### 2. char 배열을 스택으로 사용

```csharp
char[] stack = new char[number.Length];
int top = 0;
```

`top`은 다음 숫자가 들어갈 위치입니다.

즉, 현재 스택에 들어 있는 개수이기도 합니다.

---

### 3. 작은 앞 숫자 제거

```csharp
while (k > 0 && top > 0 && stack[top - 1] < digit)
{
    top--;
    k--;
}
```

`top--`은 실제로 값을 지우는 것은 아니지만, 스택의 마지막 값을 무시하게 만듭니다.

즉, 제거와 같은 효과입니다. ⚡

---

### 4. 현재 숫자 추가

```csharp
stack[top++] = digit;
```

현재 숫자를 스택에 넣고 `top`을 증가시킵니다.

---

### 5. 필요한 길이만 문자열로 만들기

```csharp
return new string(stack, 0, keepLength);
```

만약 제거 횟수가 남아 있다면 뒤쪽 숫자들은 자동으로 버려집니다.

왜냐하면 최종 길이 `keepLength`만큼만 문자열로 만들기 때문입니다. ✅

---

## ⚠️ 왜 이 코드에서 마지막 제거 반복문이 없어도 될까?

풀이 1에서는 제거 횟수가 남으면 뒤에서 제거했습니다.

하지만 풀이 2는 처음에 최종 길이를 저장합니다.

```csharp
int keepLength = number.Length - k;
```

그리고 마지막에 딱 이 길이만큼만 반환합니다.

```csharp
new string(stack, 0, keepLength)
```

따라서 뒤쪽 남는 숫자는 자연스럽게 버려집니다.

예를 들어:

```text
number = "98765"
k = 2
```

스택에는 거의 그대로 들어갑니다.

```text
98765
```

하지만 최종 길이는 3이므로:

```text
987
```

만 반환됩니다.

---

## ⏱️ 시간 복잡도

각 숫자는 한 번 들어가고 많아야 한 번 제거됩니다.

```text
O(n)
```

입니다.

## 📦 공간 복잡도

char 배열을 사용합니다.

```text
O(n)
```

입니다.

# ❌ LINQ로 풀기 어려운 이유

이 문제는 이전 숫자를 보면서 현재 숫자와 비교하고, 조건에 따라 뒤로 되돌아가며 제거해야 합니다.

즉, 상태를 계속 유지해야 합니다.

LINQ는 이런 스택형 그리디 문제에는 잘 맞지 않습니다.

억지로 LINQ를 쓰면 코드가 짧아지기보다 오히려 복잡하고 느려질 수 있습니다. 🌱

# ✅ 최종 정리

이 문제의 핵심은 다음입니다.

```text
큰 숫자가 앞에 올수록 전체 숫자가 커진다.
```

따라서 앞에서부터 보면서:

```text
현재 숫자보다 작은 앞 숫자는 제거한다.
```

를 반복합니다.

풀이 흐름은 다음과 같습니다.

```text
1. 숫자를 왼쪽부터 하나씩 본다.
2. 결과 끝 숫자가 현재 숫자보다 작으면 제거한다.
3. 현재 숫자를 결과에 넣는다.
4. k개를 제거할 때까지 반복한다.
5. 최종 길이만큼 반환한다.
```

🔢 “큰 수 만들기”는 스택과 그리디를 함께 연습하기 좋은 대표 문제입니다.
