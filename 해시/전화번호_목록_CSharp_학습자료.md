# ☎️ 전화번호 목록 — C# 학습자료

## 📌 1. 문제 핵심

전화번호 목록 `phone_book`이 주어집니다.

이때 어떤 전화번호가 다른 전화번호의 **접두어**인지 확인해야 합니다.

접두어란 문자열의 앞부분을 뜻합니다.

예를 들어:

```text
119
1195524421
```

에서 `119`는 `1195524421`의 앞부분입니다.

따라서 `119`는 `1195524421`의 접두어입니다. 🚨

문제에서 원하는 반환값은 다음과 같습니다.

```text
접두어 관계가 있으면 false
접두어 관계가 없으면 true
```

---

## ⚠️ 시간 초과가 나는 흔한 풀이

가장 먼저 떠올릴 수 있는 방법은 모든 전화번호 쌍을 비교하는 것입니다.

```text
1번 번호와 2번 번호 비교
1번 번호와 3번 번호 비교
1번 번호와 4번 번호 비교
...
```

하지만 `phone_book`의 길이는 최대 1,000,000입니다.

모든 쌍을 비교하면 시간 복잡도는 다음과 같습니다.

```text
O(n²)
```

이 방식은 거의 반드시 시간 초과가 납니다. 😵

---

# 🧠 핵심 아이디어: 정렬하면 접두어 후보가 이웃한다

전화번호를 문자열 기준으로 정렬하면 비슷한 앞부분을 가진 번호들이 가까이 모입니다.

예를 들어:

```text
["119", "97674223", "1195524421"]
```

을 정렬하면:

```text
["119", "1195524421", "97674223"]
```

이제 `119`와 `1195524421`이 바로 옆에 붙습니다.

따라서 모든 쌍을 비교할 필요가 없습니다.

```text
정렬 후 이웃한 번호끼리만 비교하면 된다.
```

이것이 이 문제의 핵심입니다. 🎯

---

# 🔍 예제 이해하기

## 예제 1

```text
phone_book = ["119", "97674223", "1195524421"]
```

정렬하면:

```text
["119", "1195524421", "97674223"]
```

이웃한 번호를 비교합니다.

```text
"1195524421".StartsWith("119") → true
```

접두어 관계가 있으므로 정답은:

```text
false
```

입니다. 🚫

---

## 예제 2

```text
phone_book = ["123", "456", "789"]
```

정렬해도 접두어 관계가 없습니다.

```text
"456".StartsWith("123") → false
"789".StartsWith("456") → false
```

정답은:

```text
true
```

입니다. ✅

---

## 예제 3

```text
phone_book = ["12", "123", "1235", "567", "88"]
```

정렬하면:

```text
["12", "123", "1235", "567", "88"]
```

`12`는 `123`의 접두어입니다.

```text
"123".StartsWith("12") → true
```

정답은:

```text
false
```

입니다. 🚨

# 🧠 풀이 1. 이해하기 쉬운 방법 — 정렬 후 이웃 비교

## 💡 아이디어

1. 전화번호 목록을 정렬합니다.
2. 앞에서부터 이웃한 두 번호를 비교합니다.
3. 뒤 번호가 앞 번호로 시작하면 접두어 관계입니다.
4. 접두어 관계가 하나라도 있으면 `false`를 반환합니다.
5. 끝까지 없으면 `true`를 반환합니다.

---

## 💻 C# 코드

```csharp
using System;

public class Solution
{
    public bool solution(string[] phone_book)
    {
        Array.Sort(phone_book);

        for (int i = 0; i < phone_book.Length - 1; i++)
        {
            if (phone_book[i + 1].StartsWith(phone_book[i]))
            {
                return false;
            }
        }

        return true;
    }
}
```

---

## 🔎 코드 해설

### 1. 문자열 정렬하기

```csharp
Array.Sort(phone_book);
```

문자열 정렬은 사전식 순서로 정렬됩니다.

예를 들어:

```text
["119", "97674223", "1195524421"]
```

은 다음처럼 정렬됩니다.

```text
["119", "1195524421", "97674223"]
```

비슷한 앞부분을 가진 번호들이 가까이 모입니다. 🧲

---

### 2. 이웃한 번호끼리 비교하기

```csharp
for (int i = 0; i < phone_book.Length - 1; i++)
```

현재 번호와 바로 다음 번호를 비교합니다.

```csharp
phone_book[i]
phone_book[i + 1]
```

---

### 3. 접두어인지 확인하기

```csharp
phone_book[i + 1].StartsWith(phone_book[i])
```

뒤 번호가 앞 번호로 시작하면 접두어 관계입니다.

예를 들어:

```csharp
"1195524421".StartsWith("119")
```

은 `true`입니다. ✅

---

## ⏱️ 시간 복잡도

정렬이 가장 큰 비용입니다.

```text
O(n log n)
```

이후 이웃 비교는 한 번만 순회합니다.

```text
O(n)
```

전화번호 길이는 최대 20이므로 문자열 비교 비용은 작습니다.

전체적으로는 다음과 같이 볼 수 있습니다.

```text
O(n log n)
```

---

## 📦 공간 복잡도

별도의 큰 자료구조를 만들지 않습니다.

```text
O(1)
```

단, 정렬 내부 구현에 따라 약간의 추가 공간이 사용될 수 있습니다.

# 🚀 풀이 2. 코드가 짧은 방법 — LINQ로 이웃 비교

## 💡 아이디어

풀이 1과 같은 정렬 기반 풀이입니다.

다만 LINQ의 `OrderBy`, `Zip`, `Any`를 사용해서 더 짧게 표현할 수 있습니다.

정렬한 배열에서 이웃한 두 번호를 묶어 비교합니다.

---

## 💻 C# 코드

```csharp
using System;
using System.Linq;

public class Solution
{
    public bool solution(string[] phone_book)
    {
        var sorted = phone_book.OrderBy(x => x).ToArray();

        return !sorted
            .Zip(sorted.Skip(1), (a, b) => b.StartsWith(a))
            .Any(x => x);
    }
}
```

---

## ✨ 코드 의미

### 1. 정렬

```csharp
var sorted = phone_book.OrderBy(x => x).ToArray();
```

전화번호를 사전식 순서로 정렬합니다.

---

### 2. 이웃한 원소끼리 묶기

```csharp
sorted.Zip(sorted.Skip(1), (a, b) => b.StartsWith(a))
```

다음과 같은 쌍을 만듭니다.

```text
sorted[0] 과 sorted[1]
sorted[1] 과 sorted[2]
sorted[2] 과 sorted[3]
...
```

그리고 뒤 번호가 앞 번호로 시작하는지 검사합니다.

---

### 3. 하나라도 접두어 관계가 있으면 false

```csharp
.Any(x => x)
```

접두어 관계가 하나라도 있으면 `true`입니다.

하지만 문제에서는 접두어 관계가 있으면 `false`를 반환해야 합니다.

그래서 앞에 `!`를 붙입니다.

```csharp
return ! ...;
```

---

## ⚠️ 프로그래머스에서 실행할 때 주의

LINQ를 사용하므로 반드시 아래 코드를 넣어야 합니다.

```csharp
using System.Linq;
```

프로그래머스 C# 제출 환경에서도 사용할 수 있습니다. ✅

---

## ⏱️ 시간 복잡도

정렬 비용이 가장 큽니다.

```text
O(n log n)
```

이웃 비교는 `O(n)`입니다.

전체 시간 복잡도는:

```text
O(n log n)
```

입니다.

---

## 📦 공간 복잡도

`OrderBy(...).ToArray()`로 정렬된 새 배열을 만듭니다.

```text
O(n)
```

입니다.

# ⚠️ 참고: HashSet으로도 풀 수 있을까?

가능합니다.

각 전화번호의 접두어를 하나씩 잘라서 HashSet에 있는지 확인할 수 있습니다.

예를 들어 전화번호가:

```text
1195524421
```

이면 다음 접두어를 확인합니다.

```text
1
11
119
1195
...
```

전화번호 길이가 최대 20이라 이 방식도 가능합니다.

하지만 이 문제에서는 정렬 후 이웃 비교가 더 간단하고 실수도 적습니다. 😊

|---|---|---|---:|---:|
| 풀이 1 | `Array.Sort` 후 이웃 비교 | 이해하기 쉽고 빠르다 😊 | O(n log n) | O(1) |
| 풀이 2 | LINQ로 정렬 후 이웃 비교 | 코드가 짧다 🚀 | O(n log n) | O(n) |

# 🏆 추천 풀이

프로그래머스 제출용으로는 **풀이 1번**을 가장 추천합니다.

이유는 다음과 같습니다.

```text
1. 코드가 짧다.
2. 메모리를 적게 쓴다.
3. 접두어 관계를 정확하게 찾는다.
4. 프로그래머스 C# 환경에서 안정적으로 실행된다.
```

LINQ에 익숙하다면 **풀이 2번**도 좋습니다.

다만 `OrderBy().ToArray()` 때문에 추가 배열이 만들어진다는 점은 알고 있어야 합니다. 📦
